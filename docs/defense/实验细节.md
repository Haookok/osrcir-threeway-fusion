## 1. 数据集与预处理

### 1.1 FashionIQ

- **来源**：https://github.com/XiaoxiaoGuo/fashion-iq
- **子集**：dress / shirt / toptee
- **原始格式**：每条 query = (reference_image_id, target_image_id, [text_caption_1, text_caption_2])
- **特殊处理**：FashionIQ 每条 query 有 2 条修改文本，按原论文做法拼接：
  ```python
  mod_text = cap1.rstrip('.') + ' and ' + cap2
  ```
- **Val 划分**：直接使用官方 `captions/cap.<subset>.val.json`
- **图库规模**：
  - dress: 3653 images
  - shirt: 6182 images
  - toptee: 5261 images

### 1.2 CIRR

- **来源**：https://github.com/Cuberick-Orion/CIRR
- **特点**：每个 query 附带一个 subset（6 张语义相关图），用于 `R_sub@K` 指标
- **Val 划分**：`captions/cap.val.json`（4181 queries，2297 image gallery）
- **注意**：test split 的 GT 不公开，需提交至 evaluation server

### 1.3 CIRCO

- **来源**：https://github.com/miccunifi/CIRCO（基于 COCO 2017 扩展）
- **特点**：
  - 每条 query 有**多个** positive targets（平均 4-5 个）
  - 图库 123K，是所有数据集中最大的
  - 指标用 **mAP@K**（而非 R@K）
- **Val 划分**：220 queries / 123K gallery
- **gt_target_names 问题**：baseline JSON 缺此字段，需从 `annotations/val.json` 补齐：
  ```python
  for q in queries:
      q["gt_target_names"] = ann[q["id"]]["gt_ids"]
  ```

### 1.4 GeneCIS

- **来源**：https://github.com/facebookresearch/genecis
- **基础**：COCO val2017 + Visual Genome
- **4 个子集**：
  - change_object（n=1960, gallery=2734）
  - focus_object（n=1960, gallery=2192）
  - change_attribute（n=2110, gallery≈10K）
  - focus_attribute（n=1998, gallery≈12K）
- **数据格式问题**：原始 attribute JSON 含 base64 图片，**文件 ~400MB**，传输易截断。做了 `slim200` 版本仅保留 id+路径用于开发调试。
- **软链接**：`datasets/GENECIS` → `/root/data/disk/datasets/genecis/`

### 1.5 图像预处理

统一用 CLIP 官方 preprocess：
```python
# CLIP ViT-L/14 preprocess
Resize(224, bicubic) -> CenterCrop(224) -> ToTensor -> Normalize(
    mean=[0.48145466, 0.4578275, 0.40821073],
    std=[0.26862954, 0.26130258, 0.27577711]
)
```

代理图分辨率：512×512（MiniMax 默认）→ 缩放到 224×224 → CLIP 编码。

## 2. 模型与 API 选择

### 2.1 MLLM：Qwen-VL-Max

- **API 提供商**：阿里云 DashScope
- **模型版本**：`qwen-vl-max`（2024-08 版本）
- **调用方式**（`src/cloudgpt_api.py`）：
  ```python
  from openai import OpenAI
  client = OpenAI(
      api_key=DASHSCOPE_API_KEY,
      base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
  )
  resp = client.chat.completions.create(
      model="qwen-vl-max",
      messages=[{"role": "user", "content": [
          {"type": "image_url", "image_url": {"url": ref_img_url}},
          {"type": "text", "text": prompt},
      ]}],
      temperature=0.0,  # 去随机性，保证可复现
      max_tokens=200,
  )
  ```
- **兼容层**：因服务器 openai 版本为 2.29.0，但 Windows 本地是 0.28.x，`cloudgpt_api.py` 写了双版本兼容，无缝切换。
- **token 估算**：每次调用 ~1500 input tokens + 100 output tokens，约 0.01 元。

### 2.2 T2I：MiniMax image-01

- **API**：`https://api.minimax.chat/v1/image_generation`
- **model 参数**：`image-01`
- **关键参数**：
  ```python
  {
      "model": "image-01",
      "prompt": D1_description,
      "aspect_ratio": "1:1",
      "response_format": "url",  # 返回 URL，过期 10min
      "n": 1,
  }
  ```
- **下载并本地缓存**：因 URL 过期，立即下载 PNG 保存到 `proxy_cache/<dataset>/<ref_img_id>.png`
- **成本**：约 0.0007 美元/张

### 2.3 视觉编码器：CLIP ViT-L/14

- **版本**：OpenAI 原版 ViT-L/14（jit 版）
- **权重位置**：`C:\Users\<user>\.cache\clip\ViT-L-14.pt`（Windows 本地）
- **统一口径**：`D1 / D2 / proxy / gallery` 全部用**同一个** CLIP 实例编码
- **避免环境差异**：gallery features 一次性在 Windows 上编码完成并缓存为 `.pt` 文件

## 3. Prompt 工程全记录

### 3.1 Baseline 复现 prompt（原论文风格）

```text
You are an expert image understanding model. Given a reference image and
a modification instruction, generate a concise description of what the
target image should look like.

Reference image: <image>
Modification: {mod_text}

Think step by step:
1. What's in the reference image?
2. What needs to be changed?
3. Output the target description (≤ 30 words).
```

### 3.2 精炼 prompt 迭代全流程

所有测试都是在 **CIRR val 200 样本** 上初步验证。

#### v1 (original)：直接对比精炼

```text
Compare the REFERENCE image and the PROXY image. Identify what's
different and update the description accordingly.
Output the refined target description.
```
**结果**：CIRR R@10 67.0 → 64.5（**−2.5**）  
**失败原因**：MLLM 把 proxy 的虚构背景写进描述，导致描述膨胀 1.9×。

#### v5：加入 CoT 推理链

```text
Step 1: Analyze reference image.
Step 2: Analyze proxy image.
Step 3: Identify gaps and produce refined description.
```
**结果**：CIRR R@10 65.5（**−1.5**）  
**失败原因**：CoT 让 MLLM "更啰嗦"，幻觉问题加剧。

#### v6：完整上下文传入

```text
You have: ref_img, proxy_img, modification, and the initial
description D1.
Generate an improved D2 by considering all of them.
```
**结果**：CIRR R@10 49.5（**−17.5，灾难性**）  
**失败原因**：信息过载，MLLM 对 D1 做了过度修改，丢失正确内容。

#### v7（关键突破）：代理图仅诊断 + 强制短描述

完整 prompt 在 `src/refine_prompts.py`，核心结构：

```text
You are analyzing TWO images:
1. A REFERENCE image (real photo)
2. A PROXY image (AI-GENERATED, may contain hallucinated details)

STRICT RULES:
- The PROXY image is used ONLY as a DIAGNOSTIC TOOL.
- DO NOT describe any details from the PROXY image (especially
  backgrounds, textures, or objects that are not mentioned in the
  modification text).
- Use the PROXY image ONLY to verify:
  * Has the requested modification been applied?
  * Is the color / object / attribute correctly changed?

Given:
- Reference: <ref_img>
- Proxy: <proxy_img>
- Modification: {mod_text}

OUTPUT:
A SHORT target description (maximum length: {len(mod_text)} characters),
focused on WHAT the target image should be.
```

**结果**：CIRR R@10 67.0 → **73.5 (+6.5)**  
**但副作用**：FashionIQ shirt R@10 30.0 → 26.0 (−4.0) —— 短描述在时尚数据上会截断关键词。

#### v9 / v10：回退尝试

- v9：V7 + "保留区分性关键词" → CIRR 提升消失（67.5）
- v10：折中 v7/v9 → 两头不讨好（66.0）

**结论**：单一 prompt 策略存在**固有矛盾**，必须靠 **Ensemble** 解决。

#### GeneCIS 专用 prompt (V2 GeneCIS)

对 GeneCIS 的短指令（如 "change man to dog"）定制：

```text
The modification is a SIMPLE, STRUCTURED edit (change/focus ×
object/attribute). Do NOT add creative details. Output must be:
"an image of [modified object/attribute]"
```

### 3.3 prompt 有效性验证

使用 `outputs/prompt_ab_test/` 目录下的 A/B 测试日志：
- 每个数据集抽 200 样本
- 记录 D1 / D2 / 两者 CLIP 特征相似度
- 对比 replaced vs ensemble 两种使用方式的效果

## 4. 三路融合完整流程

### 4.1 伪代码

```python
def three_way_fusion(ref_img, mod_text, gallery_feats, 
                      alpha=0.9, beta=0.7):
    # --- 第一轮：生成初始描述 ---
    D1 = call_mllm_baseline(ref_img, mod_text)
    
    # --- Visual Proxy 生成 ---
    proxy_img = call_t2i(D1)
    
    # --- 第二轮：V7 反幻觉精炼 ---
    D2 = call_mllm_v7(ref_img, proxy_img, mod_text)
    
    # --- CLIP 编码（并缓存） ---
    f_D1 = clip.encode_text(D1)
    f_D2 = clip.encode_text(D2)
    f_proxy = clip.encode_image(proxy_img)
    
    # --- Description Ensemble ---
    f_text = normalize(beta * f_D1 + (1 - beta) * f_D2)
    
    # --- 最终检索得分 ---
    sim_text = f_text @ gallery_feats.T
    sim_proxy = f_proxy @ gallery_feats.T
    scores = alpha * sim_text + (1 - alpha) * sim_proxy
    
    return scores.topk(K)
```

### 4.2 缓存策略（关键工程优化）

全流程三级缓存，无需重复调用 API：

| 缓存内容 | 目录 | 格式 | 大小 |
|----------|------|------|------|
| 初始描述 D1 | `outputs/baseline_desc/<dataset>.json` | JSON | 数 MB |
| 代理图 proxy | `proxy_cache/<dataset>/<id>.png` | PNG 512×512 | 数 GB |
| 精炼描述 D2 | `outputs/prompt_ab_test/<dataset>_v7.json` | JSON | 数 MB |
| CLIP 特征 | `precomputed_cache/precomputed/<dataset>_*.pt` | torch tensor | 数 GB |

**重要**：整个 9 数据集全量评估在所有缓存存在的前提下，**仅需 30 分钟**（CPU 相似度矩阵计算）。

### 4.3 特征计算细节

```python
# 标准化到单位向量 - CRITICAL
f_text = f_text / f_text.norm(dim=-1, keepdim=True)
f_proxy = f_proxy / f_proxy.norm(dim=-1, keepdim=True)
gallery_feats = gallery_feats / gallery_feats.norm(dim=-1, keepdim=True)

# Description Ensemble 必须在归一化前做加权
raw_text = beta * f_D1_raw + (1 - beta) * f_D2_raw
f_text = raw_text / raw_text.norm(dim=-1, keepdim=True)
# NOT: beta * normalize(f_D1) + (1-beta) * normalize(f_D2) —— 效果差
```

## 5. 超参数网格搜索

### 5.1 FashionIQ / CIRCO / CIRR：默认参数验证

在 200 样本上扫描 `α ∈ {0.7, 0.8, 0.9, 1.0}` × `β ∈ {0.5, 0.7, 0.9, 1.0}`：

| α \ β | 0.5 | 0.7 | 0.9 | 1.0 |
|-------|:---:|:---:|:---:|:---:|
| 0.7 | 一致差 | — | — | — |
| 0.8 | — | **提升** | — | — |
| **0.9** | — | **最优** | — | — |
| 1.0 | — | 退化 | 退化 | baseline |

**选定**：`α=0.9, β=0.7` —— 全集成、跨数据集平均最稳。

### 5.2 GeneCIS 全量网格搜索（2026-04-09）

目标：为每个 GeneCIS 子集找到最优 α/β。

**方法**：
1. 一次性预计算所有 D1 / D2 / proxy / gallery 的 CLIP 特征；
2. 在 Python 循环中遍历 `α ∈ [0.70, 0.75, ..., 1.00]` × `β ∈ [0.30, 0.40, ..., 1.00]`，每个配置只需毫秒级矩阵运算；
3. 记录 R@1/2/3 并选最优。

**结果**（全量）：

#### change_object (n=1960, gallery=2734)
| 配置 | β | α | R@1 | R@2 | R@3 | ΔR@1 |
|------|:---:|:---:|:---:|:---:|:---:|:----:|
| Baseline | 1.0 | — | 13.21 | 25.46 | 35.31 | — |
| 默认 | 0.7 | 0.9 | 13.42 | 25.51 | 36.63 | +0.21 |
| **最优 R@1** | **0.50** | **0.95** | **13.98** | 26.12 | 37.19 | **+0.77** |

#### focus_object (n=1960, gallery=2192)
| 配置 | β | α | R@1 | ΔR@1 |
|------|:---:|:---:|:---:|:----:|
| Baseline | 1.0 | — | 15.20 | — |
| 默认 | 0.7 | 0.9 | 15.26 | +0.06 |
| **最优 R@1** | **1.00** | **0.80** | **16.02** | **+0.82** |

**⚠️ focus_object 独特**：需要 **β=1.0（保留 D1）+ α=0.80（利用 proxy）**——与其他子集完全相反。

#### focus_attribute (n=1999, gallery≈12K)
| 配置 | β | α | R@1 | ΔR@1 |
|------|:---:|:---:|:---:|:----:|
| Baseline | 1.0 | — | 18.67 | — |
| 默认 | 0.7 | 0.9 | 19.27 | +0.60 |
| **最优 R@1** | **0.60** | **1.00** | **19.72** | **+1.05** |

### 5.3 网格搜索关键结论

1. **3/3 测试子集都可以通过调参超过 baseline**——证明 task-adaptive 有价值；
2. **focus_object 反向**——高 β + 低 α，与其他 3 个子集模式完全不同；
3. **最大提升**：focus_attribute +1.05 pp, focus_object +0.82 pp, change_object +0.77 pp；
4. 这是**后续工作可挖掘的消融点**：开发一个基于 query 特征的 α/β 自动选择模型。

## 6. 评估与指标计算

### 6.1 R@K（FashionIQ / CIRR / GeneCIS）

```python
def recall_at_k(scores, gt_indices, k):
    # scores: [N_query, N_gallery]
    # gt_indices: list of gt idx (1 per query)
    topk = scores.topk(k, dim=-1).indices
    hits = (topk == gt_indices.unsqueeze(-1)).any(dim=-1)
    return hits.float().mean().item() * 100
```

### 6.2 mAP@K（CIRCO）

```python
def map_at_k(scores, gt_lists, k):
    # CIRCO 每 query 多 GT, 用 average precision
    aps = []
    for q_idx, gts in enumerate(gt_lists):
        topk = scores[q_idx].topk(k).indices.tolist()
        hits = [1 if i in gts else 0 for i in topk]
        if sum(hits) == 0:
            aps.append(0.0)
            continue
        precisions = [sum(hits[:i+1])/(i+1) 
                      for i in range(k) if hits[i]]
        aps.append(sum(precisions)/len(gts))
    return sum(aps)/len(aps) * 100
```

### 6.3 R_sub@K（CIRR 独有）

每个 query 有一个 subset（6 张语义相关图），只在 subset 内做 ranking，排除无关的干扰样本，测试**细粒度区分能力**。

```python
def recall_sub_at_k(scores, subsets, gt_indices, k):
    # 对每个 query 只在它的 subset 中排序
    # subsets: list of [6 indices]
    hits = []
    for q, sub, gt in zip(scores, subsets, gt_indices):
        sub_scores = q[sub]  # [6]
        topk = sub_scores.topk(k).indices
        hits.append(int((sub[topk.tolist()] == gt).any()))
    return sum(hits) / len(hits) * 100
```

## 7. 失败尝试与教训

### 7.1 方法层面失败

| 尝试 | 结果 | 教训 |
|------|------|------|
| prompt v6（信息过载） | CIRR R@10 −17.5 | MLLM 不耐"多上下文"，应该聚焦单一任务 |
| v9/v10（prompt 折中） | 两头不讨好 | 单一 prompt 的矛盾无法通过 prompt 自身解决，需靠 Ensemble |
| 直接替换 D1→D2 | shirt −4.0 | "全或无"替换粒度太粗，需要特征空间融合 |
| Plan A 单独 α=1.0 | 退化到 baseline | proxy 确实有价值，不应完全放弃 |
| α=0.5 重 proxy | 显著退化 | proxy 质量波动大，不能作为主信号 |

### 7.2 工程层面失败 & 教训

1. **文件丢失**：  
   - `src/cloudgpt_api.py` 在某次 session 中误删，**靠 Python `.pyc` 反编译恢复**；
   - `src/refine_prompts.py` 丢失过两次，手动重建；
   - **教训**：项目第一天就应该 `git init`（我们第 X 天才做）。

2. **OOM 灾难**：  
   - Linux 服务器 3.6GB RAM 跑 CLIP ViT-L/14 直接 killed；
   - 开 9GB swap 勉强能跑，但速度慢 10×；
   - 最终迁移到 Windows RTX 4060 GPU。

3. **GeneCIS gallery 被覆盖**：  
   - change_attribute 子集的 CLIP gallery features 被一次 typo 覆盖，需要重新下载 VG 图片重建；
   - **教训**：缓存文件要 checksum + backup。

4. **OpenAI SDK 升级兼容**：  
   - Linux 服务器 openai 2.29.0（新 API）
   - Windows 本地 openai 0.28.x（旧 API）
   - 写了 `cloudgpt_api.py` 中的双版本兼容层。

5. **API 速率限制**：  
   - MiniMax 单 IP 限 60 req/min，批量跑代理图会 429；
   - 加了指数退避 + 本地队列调度。

6. **CIRCO gt_target_names 缺失**：  
   - baseline 跑出来的 JSON 没有 `gt_target_names` 字段；
   - 需要从 `annotations/val.json` 手动补齐，mAP 才能正确计算。

### 7.3 "浪费时间"但学到东西的尝试

1. **想用 SDXL 本地跑 T2I**：8GB 显存跑不动复杂 prompt，最终放弃；
2. **想改用 SigLIP 做编码器**：为对比公平性放弃，但这是很好的 future work；
3. **想 ensemble 三个 MLLM**（Qwen + GPT + Gemini）：成本 × 3，未跑；
4. **想加 re-ranking 阶段**：复杂度暴涨，单 query 耗时超 1 分钟，舍弃。

### 10.2 三路融合评分

```python
# src/visual_proxy.py
def compute_scores(f_D1, f_D2, f_proxy, gallery,
                   alpha=0.9, beta=0.7):
    # text ensemble in raw feature space
    f_text = beta * f_D1 + (1 - beta) * f_D2
    f_text = f_text / f_text.norm(dim=-1, keepdim=True)
    
    sim_text = f_text @ gallery.T
    sim_proxy = f_proxy @ gallery.T
    
    return alpha * sim_text + (1 - alpha) * sim_proxy
```

best = {}

    for alpha in np.arange(0.7, 1.01, 0.05):

        scores = alpha * (f_text @ G.T) + \
