## 〇、开场必备口径（建议熟背）

**自我介绍 / 研究主题**  
本课题复现并改进 CVPR 2025 Highlight 论文 OSrCIR（arXiv 2412.11077），研究**零样本组合式图像检索 (ZS-CIR)**。任务是：给定一张参考图像 + 一段修改文本，在大图库中检索符合描述变化的目标图，**不允许在下游数据集做任何训练**。

**一句话方法**  
我提出**三路融合 (Three-Way Fusion)** 方法：
> `score = α · sim(f_text, gallery) + (1-α) · sim(CLIP(proxy), gallery)`，  
> 其中 `f_text = normalize(β·CLIP(D₁) + (1-β)·CLIP(D₂))`，默认 `α=0.9, β=0.7`。

**一句话结果**  
在 9 个标准基准上完成全量实验，**35/35 指标全部正向提升**，主指标相对提升 5–85%，无持平、无下降。

## 一、任务理解类（几乎必问）

### Q1. 什么是 Zero-Shot CIR？为什么叫"零样本"？

**一句话**：给定 (参考图, 修改文本)，在图库中检索目标图，**且不允许在 CIR 数据集上做任何训练或微调**。

**展开**：
- 传统 CIR（如 Combiner / ARTEMIS）需要在 FashionIQ 训练集上训练 fusion 头；
- Zero-Shot CIR 只能用预训练好的通用模型（CLIP / MLLM）+ prompt，直接推理；
- 优势：**省去昂贵的人工标注三元组**；泛化到新数据集无需重训；
- 难点：没有任务特定训练信号，纯靠通用模型的语义对齐能力。

### Q2. 这个任务和传统图像检索 / VQA 有什么区别？

**一句话**：CIR 是"图+文联合 query"的**条件化检索**，既不是纯图搜图，也不是问答。

| 任务 | 输入 | 输出 | 目标 |
|------|------|------|------|
| 图像检索 | 图 | 相似图 | 全局相似 |
| 文本检索 | 文本 | 相关图 | 语义匹配 |
| **CIR** | **图 + 修改文本** | **目标图** | **组合意图** |
| VQA | 图 + 问题 | 文本答案 | 推理回答 |

### Q3. 修改文本都是什么形式？给个例子？

**一句话**：短句形式的**差异描述**，长度 5-20 词，格式多样。

**例子**：
- FashionIQ: *"is solid blue and shorter"*
- CIRR: *"has people and is outdoors"*
- CIRCO: *"the same dog but on a beach"*
- GeneCIS change_object: *"change man to dog"*

### Q4. 评价指标怎么算？R@K 是什么？

**一句话**：R@K 是 Recall@K——前 K 个检索结果里命中 ground-truth 的比例；CIRCO 用 mAP@K（有多个 GT）。

**展开**：
- **R@K** = 命中数 / 总查询数（每查询只有 1 个 GT 时）；越高越好；
- **mAP@K**：CIRCO 有多个 GT，计算 average precision；
- **R_sub@K**（CIRR 独有）：在 subset 中的 recall，排除语义干扰样本；

## 二、方法设计类（最容易深挖）

### Q5. 为什么要用 Visual Proxy？它本质是什么？

**一句话**：把 MLLM 的初始描述"画"出来，给检索补上**图像空间信号**，形成文本和图像双视角互补。

**展开**：
- 原方法全程在文本空间，描述偏差后没有恢复途径；
- proxy 是 MLLM 理解的"可视化快照"，可以暴露描述里隐式的东西（如物体布局、颜色深浅）；
- 在检索阶段，`0.1 · sim(CLIP(proxy), gallery)` 为检索提供了"图像级 anchor"；
- 50 样本消融：dress R@10 从 18 → 26（+8），确认 proxy 有效。

### Q6. 代理图不是 AI 生成的吗？会不会引入幻觉反而降低效果？

**一句话**：**正是这个矛盾花了我最多时间**——原始设计确实导致幻觉，经 4 轮 prompt 迭代用 V7 解决。

**展开**：
- 原始 prompt 让 MLLM 对比原图和代理图修正描述 → CIRR R@10 从 67 → 64.5（−2.5）；
- 诊断：MLLM 把代理图里虚构的背景、材质写进了描述，导致描述膨胀 1.9×；
- **V7 设计原则**：prompt 明确声明"代理图含幻觉细节，不得引用其新增内容"；代理图仅作为**诊断工具**（检查修改是否生效），而非描述来源；
- V7 让 CIRR 从 67 → 73.5 (+6.5)，验证了"诊断而非描述"的核心设计。

### Q7. V7 prompt 的具体内容？你能复述一下吗？

**一句话**：三条核心指令——代理图仅诊断、输出短描述、禁止引入虚构细节。

**展开（可背一下）**：
```
You are analyzing a REFERENCE image and an AI-generated PROXY image.
The PROXY image may contain HALLUCINATED details (fabricated
backgrounds, textures, objects) – DO NOT describe the proxy.
Use the proxy ONLY as a DIAGNOSTIC TOOL to verify whether the
modification text has been correctly applied (e.g. "is the color
changed?", "is the object replaced?").
Output a SHORT target description (length ≤ modification text length).
```
（具体细节见 `src/refine_prompts.py` 的 V7 模板）

### Q8. 为什么要 Description Ensemble？V7 单独不行吗？

**一句话**：V7 在某些子集（shirt R@10 −4.0）会退化，用 Ensemble **D₁ 兜底**避免退化。

**展开**：
- V7 的短描述策略在 CIRR 帮助很大，但在 FashionIQ shirt 会截断关键词；
- 本质是"全或无"地替换 D₁→D₂ 粒度太粗；
- 解决：`f_text = normalize(β·CLIP(D₁) + (1-β)·CLIP(D₂))`，β=0.7；
- β=0.7 意味着 D₁ 特征占 70% 主导，D₂ 占 30% 做增量修正；
- 最终 Ensemble 牺牲 V7 在 CIRR 的极端峰值（+6.5 → +2.0），但**换来所有 9 个子集统一正向**。

### Q9. β=0.7 这个比例怎么定的？调过吗？

**一句话**：基于 200 样本网格搜索 [0.5, 0.6, 0.7, 0.8]，0.7 在跨数据集平均最稳健。

**展开**：
- β<0.5：D₂ 主导，退化风险高（shirt 子集会负向）；
- β=1.0：退化为纯 baseline；
- β=0.7：所有 FashionIQ / CIRR / CIRCO 子集均正向；
- 对 GeneCIS 部分子集，通过 per-subset 网格搜索得到 task-adaptive 最优（如 focus_object 用 β=1.0）；
- α=0.9 同理：proxy 信号只占 10%，避免 proxy 质量波动影响主干。

### Q10. α 和 β 有什么直观含义？

**一句话**：α 控制**文本 vs 代理图**权重；β 控制**原始描述 D₁ vs 精炼描述 D₂** 权重。

| 参数 | 极值 | 意义 |
|------|------|------|
| α=1 | 100% 文本 | 退化为纯文本检索 |
| α=0 | 100% 代理图 | 退化为纯图搜图 |
| β=1 | 100% D₁ | 退化为 baseline（没有精炼） |
| β=0 | 100% D₂ | 全用精炼描述（易退化） |
| 默认 | α=0.9, β=0.7 | 文本主导 + D₁ 兜底 |

### Q11. 为什么 GeneCIS 需要专门的 prompt 和参数？

**一句话**：GeneCIS 修改文本**短而硬**（如 "change man to dog"），通用 prompt 不够；需要 task-specific 设计。

**展开**：
- GeneCIS 的 change/focus × object/attribute 四类任务非常细粒度；
- 用 V2 GeneCIS 专用 prompt（更聚焦于物体/属性变化）+ per-subset 最优 α/β；
- 网格搜索结果：各子集最优参数差异显著，如：
  - change_object: α=0.95, β=0.50
  - focus_object: α=0.80, β=1.00（**与其他反向**）
  - focus_attribute: α=1.00, β=0.60
- **这本身是一个有价值的观察点**——证明 task-adaptive 是合理的研究方向。

### Q12. 文生图模型为什么选 MiniMax？能换其他的吗？

**一句话**：在成本、速度、质量三者权衡下 MiniMax image-01 最合适；其他如 SDXL / DALL-E 3 理论可替换。

**展开**：
- MiniMax API ~0.1 元/张，2-3 秒出图，适合大批量生成；
- 试过本地 SDXL，质量相当但 8GB 显存跑不动复杂 prompt；
- DALL-E 3 质量更好但贵 10×，且国内访问不便；
- 方法对 T2I 模型**不敏感**——因为代理图只是诊断工具，细节幻觉反正要被 V7 过滤掉。

## 三、实验设置类（常被质疑公平性）

### Q13. 为什么 Baseline R@10 比原论文低这么多？

**一句话**：**MLLM 替换**（GPT-4o → Qwen-VL-Max）导致，方法本身的改进幅度与原论文不受影响。

**展开（重点口径）**：
- 原论文用 GPT-4o，本工作因国内 API 可用性限制替换为 Qwen-VL-Max；
- FashionIQ dress R@10：原论文 29.7，本工作 baseline 15.8（−13.9 pp）；
- 但 GeneCIS focus_object 本工作反而 +1.02（Qwen-VL 对短指令更好）；
- **关键：改进前后用同一 MLLM，公平可比**；
- 论文中明确说明了这一点，指导老师也确认此做法可接受。

### Q14. CIRR 和 CIRCO 数据集为什么用 val 不用 test？

**一句话**：test 集的 ground-truth 不公开，需要提交榜单；本科毕设无需榜单，用 val split 已能验证方法。

**展开**：
- CIRR test set 的 GT 在 evaluation server 上，需要提交预测；
- 与原论文 test 结果**不直接可比**，但所有方法在 val 上同样评估，公平；
- 如果上榜，预期改进趋势与 val 一致。

### Q15. 消融实验做了哪些？每个组件贡献多少？

**一句话**：三个组件独立消融在 CIRR 上分别贡献 +2.5 / +6.5 / +1.7，组合后的互补更稳健。

**展开（CIRR R@10）**：
| 配置 | R@10 | Δ | 说明 |
|------|------|---|------|
| Baseline（D₁ only） | 67.0 | — | 原论文方法 |
| + Visual Proxy（α=0.9） | 69.5 | +2.5 | 加图像信号 |
| + V7 CoT（D₁→D₂） | 73.5 | +6.5 | 精炼但风险大 |
| + V7 + Proxy | 70.5 | +3.5 | 两者叠加 |
| **Full（三路融合）** | 68.7 | +1.7 | **9 子集全正向** |

> Full 在 CIRR 单点**不是最高**，但**整体泛化最好**。这是工程上更有价值的方案。

### Q17. 实验总共花了多少 API 调用 / 多少钱？

### Q19. 原论文也用 CoT prompt，你的 V7 和他们的有什么区别？

**一句话**：原论文 CoT 用于**生成描述**；我的 V7 CoT 用于**基于代理图校验描述**。

**展开**：
- 原论文：单轮 CoT，从 (ref_img + mod_text) 直接推理出目标描述 D；
- 我的流程：**两轮 CoT**
  - 第一轮：和原论文一致，生成 D₁；
  - 第二轮：在 (ref_img + **proxy_img** + mod_text + D₁) 条件下，用 V7 反幻觉 CoT 精炼出 D₂；
- V7 的核心**增量**是**反幻觉约束**——这是原论文没有的（他们没用代理图）。

## 六、创新与局限类

### Q25. 你觉得本工作的最大创新是什么？

**一句话**：**"诊断而非描述" 的 prompt 设计原则**——把 MLLM 从自由生成器**降维为结构化校验器**。

**展开**：
- 整个领域都在让 MLLM "多说"（更多 CoT、更多细节）；
- 我反其道：让 MLLM **少说**，只做单一的"是否生效"判断；
- 这个思路可迁移到任何有 AI 幻觉风险的 pipeline；
- 同时，三路融合提供了一个可复用的**框架范式**（文本 + 图像 + 融合）。

### Q26. 方法有哪些局限性？

**一句话**：**成本、实时性、参数依赖、proxy 质量波动** 四方面。

**展开**：
1. **推理成本**：每 query 2×MLLM + 1×T2I 调用，现阶段不支持实时检索；
2. **baseline 差距**：MLLM 替换导致与原论文绝对值不可直接比；
3. **超参数依赖**：默认 (α=0.9, β=0.7) 在 GeneCIS focus_object 上次优，需 task-adaptive；
4. **proxy 质量**：T2I 对长描述 / 复杂场景仍可能生成低质图，干扰检索。

### Q27. 如果有更多时间，你会做什么？

**一句话**：**proxy 质量自适应** + **蒸馏到轻量模型** + **迭代式 proxy 生成**。

**展开**：
- **F1**：用 CLIP 相似度动态评估 proxy 质量，低质时自动降低 α；
- **F2**：把整个三路融合蒸馏到一个 CLIP 投影头，做到 10ms 级在线推理；
- **F3**：多轮 "描述→代理图→再推理" 循环，直到描述收敛；
- **F4**：提出新 benchmark 专门评估 proxy-based CIR。

## 七、"坑爹"问题预案（低概率但高风险）

### Q28. 为什么不用更强的 MLLM 如 GPT-4o / Gemini？

**一句话**：**API 可用性和成本**，Qwen-VL-Max 是国内最佳平替。

**展开**：
- GPT-4o 国内访问受限，且成本高 10×（做 12 万次调用不现实）；
- Qwen-VL-Max 质量接近 GPT-4o，且国内快速稳定；
- 方法设计对 MLLM 具体品牌**不敏感**——换成 GPT-4o 预期仍然有效；
- 这反而证明方法的**低成本可部署性**是一个实用优势。

### Q29. 你的工作 novelty 够发顶会吗？

**一句话**：三路融合的完整框架 + V7 反幻觉 prompt 是原创贡献，**有潜力发顶会 short paper / workshop**，顶会 long paper 还需强化理论分析和更大规模消融。

**展开**（诚实版）：
- 这是本科毕设，定位是**对 CVPR 论文的工程性改进**；
- 独立创新点：proxy-based CIR、V7 反幻觉原则、Description Ensemble；
- 与 CVPR long paper 相比还欠缺：理论分析、更多 baseline 对比、大规模人类评估；
- 后续可包装为 workshop / short paper 投递。

### Q30. CLIP ViT-L/14 现在有点过时了，为什么不用 SigLIP / EVA-CLIP？

**一句话**：**与原论文保持一致**是公平对比的前提；换更强编码器属于 scaling，是独立的工程方向。

**展开**：
- 原论文用 CLIP ViT-L/14，为了**严格对比**，保持相同编码器；
- SigLIP / EVA-CLIP 换上去 baseline 和改进都会一起提高，**相对改进幅度预计不变**；
- 本工作聚焦**方法学**而非**特征工程**；
- 未来工作可评估编码器升级的收益。

### Q31. 有没有人工评估？

**一句话**：**没有**——本科毕设时间有限，完全依赖数据集的客观 R@K 指标；未来工作可加。

**展开**：
- CIR 任务的 GT 是固定的，R@K 本身已经是客观人工标注的结果；
- proxy 图像的质量评估理论上需要人工，本工作用 CLIP 相似度间接衡量；
- 人工评估成本高（100 样本 × 5 人 ≈ 2000 元），超出本科毕设预算。

### Q32. 有没有失败的尝试？

**一句话**：**有很多**——prompt 迭代 4 轮、Plan A/B 两种融合方式、v9/v10 后续回退尝试。

**展开**（挑 1-2 个讲）：
- **prompt v6 灾难**：把 D₁ 和代理图一起传给 MLLM，让它"综合考虑"——CIRR R@10 **从 67 跌到 49.5 (−17.5)**，信息过载导致 MLLM 过度修改；
- **后 V7 的 v9 v10 尝试**：想兼顾 CIRR 的提升和 shirt 的不退化，两个都折衷，结果两头不讨好（R@10=66.0 < baseline）；
- 这些失败反而帮助理解：**简单 ensemble 比复杂 prompt 鲁棒**。

## 八、临场技巧 tips

1. **被问到不熟悉的指标/概念**：先承认不熟，再用自己掌握的类比解释（如"我没直接对比这个，但根据我的数据，预期趋势是……"）。
2. **老师质疑 baseline 差距**：直接用 Q13 的标准话术，强调改进前后同 MLLM、相对提升可比。
3. **被问到 V7 prompt 原文**：可以说"具体文本在 `refine_prompts.py` 里，核心三条指令是……"——无需逐字背。
4. **如果老师连续追问细节，时间超过 2 分钟**：主动说"这里涉及多个细节，如果需要我可以在答辩后给您发详细文档"（你有 `答辩QA预案.md` 和 `实验细节.md` 可以发）。
5. **每个答案结束用"欢迎指正"**——礼貌且留余地。
