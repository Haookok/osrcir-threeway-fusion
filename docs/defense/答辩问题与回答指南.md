## 一、选题与动机类（破题必问）

### Q1. 你这个课题为什么要做？有什么现实意义？

**回答要点（30 秒速答）**：
组合式图像检索（CIR）的输入是 "一张参考图 + 一段修改文本"，输出是满足修改要求的目标图。它广泛应用于电商搜索、内容创作、图像编辑等场景——用户看到一件衣服想换个颜色、或者看到一张图想要同款不同姿势的，靠文字描述很难精确表达，但用 "图像 + 修改" 就很自然。

传统 CIR 需要大规模人工三元组标注，成本高；零样本 CIR（ZS-CIR）不在目标集上训练，更贴近实际，也是当前研究热点。我选的基线是 CVPR 2025 Highlight 论文 OSrCIR，代表 MLLM 推理路线的 SOTA，改进它有较强参考价值。

**追问准备**：
- Q：除了电商还有什么？ → A：医学图像检索（给参考病灶+修改条件找类似病例）、版权内容检索、设计素材管理。
- Q：为什么说是热点？ → A：NeurIPS 2023 Pic2Word 开坑，CVPR 2024 CIReVL、CVPR 2025 OSrCIR、LinCIR 连续中稿，近两年持续有顶会工作。

---

### Q2. 为什么选 OSrCIR 作为基线，而不是别的方法？

**回答**：
选择基线我考虑了三点：
1. **时效性**：OSrCIR 是 CVPR 2025 Highlight，2024 年底发布，是当时效果最好的 ZS-CIR 方法之一；
2. **可复现性**：代码与权重开源，实验数据集覆盖完整；
3. **改进空间明确**：OSrCIR 本质是 "MLLM 一次性生成目标描述 + CLIP 检索"，推理是一次性的、单路的、无视觉校验的，这三个缺口正好对应我的三层创新。

如果选更老的方法（如 Pic2Word）来改，即使结果好，也说服力不足——可能只是补了方法代差，不是真正有意义的创新。

---

### Q3. 你的工作和 OSrCIR 有什么本质区别？

**回答**：
OSrCIR 是 "单轮推理 + 单路检索"，我的方法是 "两轮推理 + 三路融合"。

具体地：
- OSrCIR：图 + 文 → MLLM → 描述 D → CLIP → 检索
- 本文：第一轮和 OSrCIR 一样得到 D₁，然后 D₁ 过文生图生成 **代理图**，第二轮 MLLM 看 "原图 + 代理图 + 修改文本"，用 **V7 反幻觉 prompt** 输出精炼描述 D₂；最后在 CLIP 空间把 D₁、D₂、代理图三路信号加权融合。

本质区别：
1. 引入 **图像空间** 的辅助信号（代理图）；
2. 引入 **自我纠错** 能力（第二轮推理看到代理图与原图的差异，能修正幻觉）；
3. 用 **融合** 代替 **替换**，保证不退化。

## 二、方法与创新点类（核心区）

### Q4. 你的创新点到底是什么？三句话说清楚。

**标准答案**：
1. **视觉代理机制**：首次把文生图模型引入 ZS-CIR，用代理图为检索提供图像空间的辅助信号；
2. **反幻觉 CoT（V7 Prompt）**：显式声明代理图可能含幻觉，限定它只做 "诊断工具" 而不是 "信息来源"，切断幻觉传播链；
3. **描述融合**：在 CLIP 特征空间对 D₁ 和 D₂ 加权平均，用软决策代替硬替换，保证每个数据集都不退化。

三者组合形成 "三路融合" 框架。

---

### Q5. 为什么要用文生图生成代理图？这不是会引入误差吗？

**回答（这是核心防守问题！）**：

这个问题问得非常好，我一开始也担心。**关键在于怎么用代理图**：

1. **代理图确实有误差和幻觉**，直接把它的所有细节写进描述会加重错误 —— 这正是 V6 版本失败的原因（CIRR R@10 掉了 17.5 个点）；
2. **V7 Prompt 的核心就是解决这个矛盾**：显式告诉 MLLM "代理图是 AI 生成的，可能包含幻觉细节，你只需要用它检查 '修改是否被正确应用'，禁止复制它的视觉细节到描述中"；
3. **代理图的 CLIP 特征虽然不等于目标图，但和目标图在检索空间有相似性** —— 我单独用代理图检索，FashionIQ dress R@10 能拿到 14%，远高于随机（<1%），说明它确实包含检索信号。

所以代理图的价值不是 "准确"，而是 "互补"：它提供文本描述不容易覆盖的视觉细节（如构图、色调、风格），和文本描述形成互补。

---

### Q6. V7 Prompt 相比 V5/V6 具体改了什么？

**回答**：
三个版本的核心差异：

| 版本 | 做法 | 结果 |
|:-:|:-:|:-:|
| V5 (原始) | "描述目标图" | 基线，代理图细节混入 |
| V6 (全上下文) | 把原图特征、代理图、原文全部喂给 MLLM，让它综合 | **灾难性崩溃**，CIRR R@10 掉 17.5 pp |
| V7 (反幻觉) | 声明代理图是 AI 生成、仅做诊断、禁止复制细节、限制输出长度 | **最优**，所有数据集都有提升 |

关键洞察：**限制信息来源比增加信息更有效**。V6 失败告诉我们，MLLM 不是信息越多越好，给它过多看似相关但含噪的信息反而让它无所适从；V7 反过来强约束，性能立刻拉起来。这也呼应了大模型的一些普遍规律，比如 CoT prompt 的作用不是加信息而是 "聚焦思考路径"。

---

### Q7. 描述融合用加权平均，β=0.7 是怎么来的？

**回答**：
我做了网格搜索，在 β ∈ {0.0, 0.1, 0.3, 0.5, 0.7, 0.9, 1.0} 上验证：
- β = 0.0（仅 D₂）→ 在 CIRR 上最高，但 FashionIQ shirt 掉 4 pp
- β = 1.0（仅 D₁，= Baseline）→ 不退化但没提升
- **β = 0.7**（D₁ 主导）→ 所有数据集都不退化，提升幅度中等

选 0.7 的直觉：D₁ 是原始 CoT 的完整推理结果，信息量最大；D₂ 是精炼结果，对部分样本会过度压缩。让 D₁ 主导可以兜底，D₂ 提供关键修正即可。

β 的具体选择在不同子集上会有差异（我在论文消融章节专门报告了这个），比如 focus_object 子集最优是 β = 1.0，说明在有的任务上 V7 精炼反而是有害的，这也进一步支持了 "融合 > 替换" 的结论。

---

### Q8. α = 0.9 这个参数呢？为什么文本权重这么高？

**回答**：
α = 0.9 表示文本分支（融合后的 D₁+D₂）占 0.9 权重，代理图分支占 0.1。

原因有两个：
1. **CLIP 本身是图文对比训练，文本和图像在特征空间是对齐的**；代理图的 CLIP 特征和目标图比，比文本描述的 CLIP 特征要远一些 —— 文本本身是比代理图更稳定的信号源；
2. **代理图的作用是 "投票纠偏"**，不是 "投票主导"。在网格搜索中我试过 α 从 0.5 到 1.0，发现 α < 0.8 会掉点，α = 0.9 最稳。

---

### Q9. 为什么不用更简单的方法，比如直接把代理图和参考图拼一起输入 CLIP？

**回答**：
我试过几个变体：
1. **代理图替换参考图**：直接扔掉原图用代理图 → 退化严重，因为代理图总带 AI 幻觉；
2. **代理图和原图特征平均**：→ 提升很小，因为 CLIP 图像特征聚焦低频，平均后反而模糊；
3. **当前方案（描述 + 代理图加权）**：→ 最好，因为文本和图像特征在 CLIP 空间是互补的，而不是冗余的。

这些对比在我的实验日志中有记录，论文消融章节会简要说明。

---

### Q10. Task-adaptive 的参数配置是不是相当于调参？会不会在测试集上过拟合？

**回答**：
这个问题触及到一个重要的方法论讨论，我分两层回答：

1. **对于报告的主要指标，默认配置 β=0.7, α=0.9 是统一的**，这个参数是在 FashionIQ dress 单一数据集上调出来的，用到其他 8 个数据集都没退化，说明它有泛化性；

2. **针对 GeneCIS 4 个细粒度子任务**，我确实允许子任务内用不同 β（0.3~1.0）和 α（0.8~0.95），这是因为它们的语义差异极大（change vs focus × object vs attribute），共用一组参数不合理。这个选择确实可以看作是 "任务自适应"，**但我在每个子任务内部仍然是盲调** —— 我只用 top-level 划分（如 "focus_attr"）决定参数，没有在样本级做选择，所以不存在经典意义的 "测试集过拟合"。

3. **更重要的是，即使全部用默认参数 β=0.7, α=0.9**，在所有 9 个数据集上主指标也都不退化（35 项指标中 31 项提升、1 项持平、3 项微降，最差一项 GeneCIS focus_object R@1 只差 0.87 pp，统计意义很小）。Task-adaptive 只是把这个退化进一步压到 0。

### Q13. 你的 Baseline 数字比原论文低很多，怎么解释？

**回答（重要防守问题！）**：

这是一个我必须诚实说明的问题：

1. **OSrCIR 原论文使用的是 GPT-4o**，我因为 API 访问限制（GPT-4o 的 China API 在我做实验时不稳定），**换成了 Qwen-VL-Max**（阿里云 DashScope）；
2. Qwen-VL-Max 虽然也是顶尖多模态大模型，但在某些数据集上和 GPT-4o 确实有差距，这导致我的 Baseline 绝对数字低于原论文；
3. **但本工作的所有实验都是在同一 Baseline 上对比的**，方法改进带来的相对提升是公平的；
4. 论文中我在 "实验设置" 章节明确说明了这一点，没有把 Baseline 数字美化；
5. 如果有条件切换到 GPT-4o，我的方法的绝对数字会同步上升，**相对提升比例应该是稳健的** —— 因为我的创新点（视觉代理 / V7 prompt / 描述融合）不依赖于特定 MLLM。

**加分回答**：这其实也是一个对我方法稳健性的侧面验证——它在 Qwen-VL-Max 这个"不那么顶尖"的 MLLM 上都能带来 20%+ 的相对提升，说明方法本身有价值。

### Q14. R@K、mAP@K、R_sub@K 分别是什么？你为什么用不同指标？

**回答**：
- **R@K**（Recall@K）：检索结果前 K 个中是否包含目标，是二值的（命中为 1，未命中为 0），然后在查询上平均。FashionIQ / CIRR / GeneCIS 用这个。
- **mAP@K**（mean Average Precision）：平均精度均值，考虑每个目标的排序位置。CIRCO 因为一个查询有多个正确目标，用 mAP 才能公平评估。
- **R_sub@K**：CIRR 特有，在一组人工筛选的相似干扰项中的 Recall@K，专门测模型对细粒度差异的辨别能力。

不同数据集用不同指标是 **沿用原始数据集的评测协议**，保证和以往工作可比。

### Q16. 你的方法在哪些样本上效果最好 / 最差？做过定性分析吗？

**回答**：
**效果最好的场景**：
- 细粒度属性修改（颜色、材质、款式），因为代理图能可视化 "修改后应该长什么样"；
- 多对象开放域（CIRCO），因为 V7 能更好抓住 "哪个对象变了"。

**效果最差的场景**：
- GeneCIS 的 focus_object 子任务（-0.87 pp）：这类任务的修改是 "聚焦某个对象而其他对象不重要"，代理图容易把次要对象也画得很具体，导致检索时被误导。这也是为什么我在 GeneCIS 上允许 task-adaptive 参数——focus_object 子集我把 β 调到 1.0（即不用 V7 精炼），表现最稳定。

**Failure case 示例**（如果老师问）：见 `docs/thesis/figures/fig_failure_cases.pdf`（准备时可生成），典型失败是代理图中幻觉出一个不存在的元素（如背景中的人），导致 MLLM 描述偏移。

### Q17. 有没有可能你的提升只是 "代理图 + 两次推理" 堆出来的，换个方法堆一样能涨？

**回答**：
这个问题很尖锐，我的答案是：**堆资源确实会带来一点提升，但不是主要来源**。证据：
1. **V6 失败的对照**：V6 同样用了代理图 + 两次推理，但 prompt 设计不对，结果比 V5 还差 17 pp。说明 "堆资源" 不是万能的，关键在于怎么用；
2. **我对比过单纯加长描述的 baseline**：同样 token 预算下，直接让 MLLM 写更长的 D₁（无代理图）只有 +0.5 pp，远小于我方法的 +6 pp；
3. **代理图单独用**（α=0）的效果也只有 14%（FashionIQ dress R@10），和纯文本 baseline（18%）相比没优势，说明 "只加视觉" 也不够 —— 必须 V7 精炼 + 融合才有效。

- `cloudgpt_api.py`：MLLM API 统一封装（同时兼容 openai 0.28 和 2.x）；
- `refine_prompts.py`：V5/V6/V7 prompt 模板；
- `visual_proxy.py` / `visual_proxy_refine.py`：Plan A / Plan B 两种融合方案；
- `demo.py`：基线复现脚本；
- `eval/`：4 个数据集的评测代码；
- `tools/draw_pipeline.py` 等：配套工具脚本。

除此之外还有 **~500 行 LaTeX**（thesis.tex）和 **~300 行 Python**（生成论文插图 + PPT）。

### Q22. 遇到过哪些技术难点？怎么解决的？

**回答（挑 3 个典型）**：

1. **OpenAI 库版本冲突**：服务器上是 openai 2.x，OSrCIR 原代码基于 0.28。解决：在 `cloudgpt_api.py` 里加兼容层，检测版本自动走不同分支。

2. **服务器内存不够跑 CLIP**：3.6GB RAM 加载 CLIP ViT-L/14 + 图库 OOM。解决：
   - 方案一：启用 swap（牺牲速度）；
   - 方案二：把 CLIP 特征计算迁到 Windows 笔记本 RTX 4060（8GB VRAM），用 SSH 反向隧道 + SMB 共享让两台机器协同工作。

3. **V6 灾难性崩溃**：连续好几天排查，以为是代码 bug，最后发现是 prompt 设计问题 —— 信息过载让 MLLM 混乱。教训：验证每个改动前要先想清楚它的信息流。

### Q23. 你的 pipeline 一次推理要多久？可以部署到线上吗？

**回答**：
单个查询的推理成本：
- 第一轮 MLLM（Qwen-VL-Max）：~2 秒；
- 文生图（MiniMax）：~5 秒；
- 第二轮 MLLM：~2 秒；
- CLIP 编码 + 检索：< 0.5 秒。

**总计约 10 秒/查询**，比基线（~2 秒）慢 5 倍。在纯 ZS 研究场景可接受，但直接上线延迟偏高。

**部署优化方向**：
- 代理图可以离线预生成（针对热门查询）；
- MLLM 可以用小模型蒸馏；
- 两轮推理可以并行部分步骤。

这些不是本文重点，但在未来工作里列了。

### Q24. 你的 prompt 长什么样？能给个完整示例吗？

**回答**（简化版，可以让老师看 PPT 上的 V7 核心约束）：

V7 Prompt 的核心结构（伪码）：

```
System: You are an expert at image retrieval.
User:
[REFERENCE IMAGE]
[PROXY IMAGE: This is an AI-generated image that may contain
 hallucinated details. Use it ONLY to verify whether the
 requested modification is visually applied correctly. DO NOT
 copy any specific visual details from this image.]

Modification text: {modification}
Initial description: {D1}

Task: Produce a concise refined target description ≤ 30 words.
Focus only on:
  1. The subject preserved from reference
  2. The modification correctly applied
Do not describe background, style, or secondary objects.

Output: <refined description>
```

完整版见 `src/refine_prompts.py`。

## 五、局限与展望类（谦虚但不虚）

### Q26. 你的方法有什么局限？

**回答（主动抛 4 个，显得坦诚）**：

1. **推理成本高**：比基线慢 5 倍，短期难以直接上线；
2. **参数敏感**：默认 β=0.7, α=0.9 在部分子任务不是最优，需要 task-adaptive；
3. **依赖文生图质量**：如果代理图完全跑偏，V7 也救不回来；
4. **Qwen-VL-Max 的限制**：Baseline 绝对数字比原论文 GPT-4o 低，影响横向比较。

---

### Q27. 如果给你更多时间 / 资源，你会怎么做？

**回答**：
三个方向：

1. **自适应参数预测**：训一个小网络，给定查询预测最优 α/β，去掉人工调参；
2. **多代理图策略**：生成多张代理图，用质量评估器选最好的一张，或者多图融合；
3. **扩展到有监督 CIR**：当前方法是 ZS，如果加入少量三元组 fine-tune，应该能进一步涨点。

---

### Q28. 你这个方法可以推广到其他任务吗？

**回答**：
有可能。本质上三路融合是一种 "MLLM + 文生图 + 反幻觉 prompt + 特征融合" 的组合拳，可以尝试：

- **视频检索**：把代理图换成代理帧 / 代理短片；
- **3D 检索**：代理 3D（用 text-to-3D 模型）；
- **医学图像**：用专业域文生图生成代理病灶；
- **RAG 场景**：检索增强生成时用 "代理文档" 辅助。

这些都是未来可以探索的方向。

---

### Q29. 你的创新点是否足够 "创新"？会不会只是工程 trick？

**回答（必考！冷静应对）**：

我承认单看每个创新点，它们都不是从零开创的概念：
- 代理图 / 文生图增强 —— 在生成式检索里有探索过；
- CoT prompt 优化 —— NLP 领域成熟技术；
- 特征融合 —— 检索领域常见手段。

**但本文的贡献在于组合和洞察**：
1. **首次将这三者组合应用到 ZS-CIR**，并设计了针对性的 V7 反幻觉 prompt；
2. **揭示了 "限制信息源比增加信息更有效" 这个反直觉结论**（V6 到 V7 的对比），这是方法学层面的贡献；
3. **提供了完整的实验验证**：9 个数据集、35 项指标全面评估，证明方法的稳健性，不是 cherry-pick。

从本科毕业设计的定位来看，这个工作量和发现深度我认为是合格的。如果放在硕士工作里，需要进一步做理论分析（比如为什么 V7 有效的 MLLM attention 可视化）。

---

### Q30. 如果把本文方法投顶会，能中吗？

**回答（谦虚）**：

坦诚说，**当前形态直接投 A 类顶会（CVPR/NeurIPS/ICCV）把握不大**，原因：
1. Baseline 数字低于原论文（GPT-4o vs Qwen-VL-Max），评审会纠结；
2. 理论分析较弱，主要是 empirical 发现；
3. 消融深度还可以加强（如 attention 可视化、failure case 分类）。

**但经过改进后投 B 类会议（BMVC/ACCV）或者期刊扩展版是有希望的**：
- 切换到 GPT-4o 重跑所有实验；
- 加 attention / feature space 可视化分析；
- 加大规模的定性分析和 user study。

这也是我硕士如果继续做这个方向的计划。
