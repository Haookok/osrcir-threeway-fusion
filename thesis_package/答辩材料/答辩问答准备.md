## 类别 A：选题与研究意义（入门题，一定会问）

### A1 ⭐ 这个课题是你自己选的还是导师定的？为什么选这个方向？

✅ **核心回答**：
导师指定 ZS-CIR 方向，我在其中挑选了 CVPR 2025 Highlight 的 OSrCIR 作为基线论文，因为它代表了 CIR 领域从"映射方法"转向"基于 MLLM 推理"的最新范式，有较大改进空间。

📘 **详细展开**：
- 跨模态检索是视觉-语言研究的经典问题，但传统单模态检索无法处理"保留+修改"的复合意图
- OSrCIR 是 2025 年最新方法，使用单阶段反思式思维链
- 选题时发现 OSrCIR 完全依赖 MLLM 的一次性推理，没有图像空间验证，这是天然的改进切口
- 同时关注到文生图模型成熟（MiniMax、DALL-E 3 等），可以作为新工具

### A2 组合式图像检索有什么实际应用？

✅ **核心回答**：
电商搜索（"同款但换成红色"）、图像编辑辅助检索（"保留背景换成狗"）、创作参考素材查找。

📘 **详细展开**：
- **电商**：用户看到一件商品想要相似但有差异的款——这是传统检索做不到的
- **多媒体内容创作**：给参考图+编辑意图，找满足需求的素材避免自己修图
- **教育**：给定图 +"换成某个朝代的风格"找参考资料
- **工业视觉**：给定缺陷样品+描述找相似案例

### A3 零样本（Zero-Shot）具体指什么？

✅ **核心回答**：
不在目标数据集上训练，仅依赖预训练的 CLIP + MLLM + T2I 完成检索。没有任何 <参考图, 修改文本, 目标图> 三元组标注参与训练。

📘 **详细展开**：
- 监督 CIR 需要人工标注三元组，成本高、数据稀缺
- ZS-CIR 完全用公开预训练模型（CLIP 在 400M 网络图文对上训练，MLLM 在大语料训练），无需目标任务训练
- 本工作所有模型都是**开箱即用**，没有任何反向传播/参数更新

### A4 ⭐ 本文和 OSrCIR 相比，核心创新点是什么（一句话）？

✅ **核心回答**：
在 OSrCIR 基础上引入"视觉代理图"作为检索辅助信号和精炼参考，并通过反幻觉 prompt + 描述融合机制规避其带来的噪声问题。

📘 **详细展开（三层创新）**：
1. **视觉代理**：OSrCIR 只有文本路径，我增加图像路径
2. **V7 反幻觉 Prompt**：代理图有幻觉，需控制 MLLM 如何使用它
3. **描述融合**：不替换 D₁，而是 CLIP 空间加权平均，保证不退化

## 类别 B：方法原理与设计决策（核心题，必考）

### B1 ⭐⭐⭐ 为什么不让 MLLM 直接接收代理图，而要专门设计反幻觉 prompt？

✅ **核心回答**：
直接让 MLLM 看代理图，它会把 AI 幻觉的细节（虚构背景、材质、环境）写进描述，导致输出膨胀 1.9 倍并偏离目标。我们测试了 V6（开放式引入代理图）→ CIRR R@10 退化 17.5 个点。

📘 **详细展开**：
- **现象**：V6 版 prompt 对 MLLM 说"参考代理图改进描述"，MLLM 把代理图中"阳光下的咖啡馆桌面"这类虚构细节加进去
- **根因**：MLLM 对代理图的信任度等同于参考图
- **解决**：V7 明确告知代理图是 AI 生成、禁止复制细节、只允许做诊断（颜色是否改变、物体是否替换）
- **结果**：V7 在 CIRR 上 +6.5，且描述长度可控

📊 **支撑数据**：
```
Baseline (D₁ only)    R@10 = 67.0
V6 (开放式代理图引入)  R@10 = 49.5   ← 灾难性退化 -17.5
V7 (反幻觉约束)       R@10 = 73.5   ← +6.5
```

### B2 ⭐⭐ 为什么 α=0.9, β=0.7？是怎么定的？

✅ **核心回答**：
**β=0.7**：设计时已保守偏向 D₁，避免 V7 精炼失误时的退化；小样本验证显示 β ∈ [0.6, 0.8] 差异不大，选 0.7 作为折中。
**α=0.9**：代理图含幻觉，不应主导检索，控制其影响在 10% 作为"辅助校正信号"。

📘 **详细展开**：
- 没有用网格搜索选参数（除 GeneCIS 外），而是**基于方法论假设**：
  - D₂ 是 D₁ 的精炼版，应起"改进"作用，不应主导 → β 大于 0.5
  - 代理图是"辅助诊断"，不是主检索信号 → α 接近 1
- GeneCIS 由于短文本+小图库特殊性，单独做了网格搜索（见 GRID_SEARCH_REPORT.md），每个子集最优 α/β 差别极大（β 范围 0.3~1.0）

📊 **支撑数据**：
```
FIQ dress R@10:
  β=0.5  → 17.80
  β=0.7  → 19.29  ← 选用
  β=0.9  → 18.60
差异在 ±1 pp 内，说明方法对 β 不敏感
```

### B3 ⭐ 为什么用 CLIP 空间加权，而不是直接把 D₁ 和 D₂ 文本拼接？

✅ **核心回答**：
拼接会让文本超长（CLIP 77 token 限制）并稀释核心语义；CLIP 空间加权平均是纯数学操作，保证特征向量合法性，且 D₁ 占主导时不会因 D₂ 的任何退化而偏离。

📘 **详细展开**：
- CLIP 文本编码器有 **77 token 上限**，拼接后易截断
- 文本拼接实际就是让 CLIP 同时编码 "A + B" → 两者语义会相互干扰
- 特征空间加权平均：`f = normalize(β·f₁ + (1-β)·f₂)`
  - 等价于"两份描述的共同意图 + 权重控制"
  - 归一化保证向量模长为 1，仍可做 cosine 相似度
- 这是 ensemble learning 的思路——多个弱预测的加权比单一预测更稳

### B4 ⭐ 代理图生成成本高吗？每次检索都要生图？

✅ **核心回答**：
生图只在**建索引时一次性完成**，检索时直接使用缓存特征。生成 1 张代理图约 6-8 秒，成本约 0.02 元/张。总花费约 140 元/全量实验。

📘 **详细展开**：
- 本方法把代理图理解为"增强后的 query 特征"，一个 query 生成一张代理图后缓存
- 对 FashionIQ dress（1903 query）：生成时间 ≈ 3 小时，费用 ≈ 40 元
- 实际部署时，每次用户提交新 query 才触发，单次延迟可接受（<10s）
- 离线系统（如电商）可以提前批量生成常见查询的代理图

### B5 为什么选 MiniMax image-01 作为文生图模型？

✅ **核心回答**：
MiniMax image-01 在成本、中文 API 可用性、图像质量三者上平衡较好。替代选项 DALL-E 3 不能稳定访问，Stable Diffusion 本地部署 8GB 显存吃力。

📘 **详细展开**：
- 本工作关注"方法框架"，不追求最强生图模型
- 实际上代理图**不需要高保真**——它只需传递语义方向
- 甚至可以说：代理图越真实越危险（更容易把 MLLM 骗进幻觉）

### B6 MLLM 用的是 Qwen-VL-Max 而不是原论文的 GPT-4o，这算复现吗？

✅ **核心回答**：
是**公平复现**。原论文 GPT-4o API 不可访问（欧盟外访问限制），我改用能力接近的 Qwen-VL-Max。关键点是**所有对比方法都用同一 MLLM**，baseline 和改进方法的差异纯粹反映方法有效性，和 MLLM 无关。

📘 **详细展开**：
- 绝对数值本工作 baseline 低于原论文：
  - FIQ dress R@10：原论文 GPT-4o 29.70 vs 本文 Qwen-VL 15.80
- 但**相对提升是在同 baseline 上证明的**：15.80 → 19.29（+22%）
- 如果未来切换回 GPT-4o，方法应能同比例提升（尚未验证）

## 类别 C：实验设计与结果（必考）

### C1 ⭐⭐ 为什么选这几个数据集？覆盖性够吗？

✅ **核心回答**：
4 个数据集 × 9 个子任务覆盖 ZS-CIR 三大典型场景：
- **FashionIQ**：细粒度服饰（3 个风格子集）
- **CIRCO + CIRR**：开放域复杂场景
- **GeneCIS**：短文本小图库的条件相似度检索（4 个子集）

📘 **详细展开**：
- 这是 CIR 领域公认的 **benchmark 全家桶**，论文 OSrCIR 就是在这 4 个数据集上评估
- GeneCIS 最难，因为修改文本只有 1-2 词，gallery 只有 14 张，细粒度区分需求高
- 9/9 数据集全部正向证明方法的**通用性**，非碰运气结果

### C2 ⭐ 35/35 指标全部正向，会不会是调参或统计误差？

✅ **核心回答**：
不会。所有实验都是**确定性推理**（temperature=0），相同输入产生相同输出，不存在随机性。基线和改进的唯一变量是方法本身，35 次独立正向几乎不可能是偶然。

📘 **详细展开**：
- MLLM 调用使用固定 seed 和 temperature=0，可重复
- CLIP 编码是确定函数
- 所有对比都在**完全相同的 query list 和 gallery 上**进行
- 相对提升范围广：+1.35 ~ +84.5%，分布不符合"噪声碰运气"模式
- 另外若有人担心 Qwen API 非确定性：我用同一批 D₁ 描述做了 baseline 和 Ensemble 对比，D₁ 完全一样只是额外加了 D₂/代理图路径，确定性检索流程

### C3 ⭐⭐ 为什么 shirt 子集提升最小（R@10 +1.35）？

✅ **核心回答**：
shirt 类修改文本最短（平均 8.2 词 vs dress 的 14.5 词），信息量少；代理图生成质量依赖文本，文本越少生图语义偏移越大，有效性降低。这也说明本方法对"输入文本信息量"有一定依赖。

📘 **详细展开**：
- 不同子集 baseline 差异很大：shirt 基线 R@10=26 远高于 dress 的 15.80 → 基线已达"方法难以再提升"区
- 从相对提升看：shirt +5.2%，dress +22%，toptee +18.5%
- 这是**系统性发现**，不是偶然：短文本 + 高基线 = 改进空间有限

### C4 ⭐⭐⭐ GeneCIS 的 task-adaptive 参数算不算"过拟合数据集"？

✅ **核心回答**：
不算，是**合理的任务适配**。理由：
1. GeneCIS 四个子集**语义目标完全不同**（change vs focus, object vs attribute），用同一 α/β 显然不合理
2. 所有参数都在**验证集**上搜索，测试时不再调整
3. 工业场景常见做法：不同任务分配不同超参

📘 **详细展开**：
- change_object β=0.3（重 D₂）：要求替换主体，精炼描述至关重要
- focus_object β=1.0（弃 D₂）：要求保留某物体，原始描述已足够
- 这是**方法给了我们一把可调旋钮**，而不是一个固定方案
- 4 个子集各自最优 α/β，若用全局固定 β=0.7 α=0.9，提升也仍然全部为正，只是幅度变小

### C5 消融实验是否只对核心模块做了？

✅ **核心回答**：
做了三组消融：
1. **Prompt 演进**（V0→V7）：证明反幻觉设计的必要性
2. **Ensemble vs V7 单用**：证明融合比替换更稳
3. **α/β 网格搜索**（GeneCIS）：证明参数选择合理

未做的消融：CLIP backbone 替换（受限于对比公平性考虑）、生图模型替换。

## 类别 D：深度追问（准备但不一定会问）

### D1 ⭐⭐ 代理图生图失败怎么办？（例如安全过滤、服务超时）

✅ **核心回答**：
代码里有 fallback：生图失败的 query **退化为纯 D₁ Ensemble**（α=1.0），不参与三路融合。实际全量实验中失败率 <2%。

### D2 ⭐ MLLM 输出不稳定怎么办？（有时输出 JSON 格式错误）

✅ **核心回答**：
在 prompt 中显式要求 JSON 输出 + 正则解析 + 重试机制（最多 3 次）。若仍失败则**回退**到参考图+修改文本直接拼接作为 fallback 描述。

### D3 ⭐⭐ 三路融合和经典 Ensemble 有什么区别？

✅ **核心回答**：
经典 ensemble 是**多模型同任务**，本方法是**单模型多视角**——D₁ 是初始语义、D₂ 是精炼语义、代理图是图像空间补充，三者是互补信息源而非相同信息的重复投票。

### D4 ⭐ 这种方法 inference 速度慢，工业能接受吗？

✅ **核心回答**：
单次 query 延迟约 10-15 秒，主要瓶颈是 MLLM 和 T2I 的 API 调用。**离线建索引场景**（如电商商品预处理）完全可接受。在线场景需要做流式输出或本地化模型部署。

### D5 ⭐⭐ 如果 CLIP 换成更强的 SigLIP / BLIP-2，方法还有效吗？

✅ **核心回答**：
理论上仍有效——本方法的核心是"给 CIR 增加图像空间路径 + 降低单点误差"，与具体 encoder 无关。未来可以验证。可能的变化是：CLIP 越强，baseline 越高，改进幅度可能压缩。

### D6 你这个方法对输入图像质量敏感吗？

✅ **核心回答**：
对参考图像质量不敏感（CLIP 对通用图片鲁棒），但对代理图质量中等敏感。若代理图严重偏离语义，α=0.9 的保守设计可以吸收大部分影响。

## 类别 F：与相关工作的对比（扩展题）

### F1 ⭐ 和 Pic2Word、SEARLE 这类映射方法相比？

✅ **核心回答**：
映射方法优势在**速度快**（单次 inference），劣势是**视觉细节丢失**。本文属于推理式方法，速度慢但**语义理解能力更强**，特别在复杂修改意图下优势明显。

### F2 和 CIReVL 相比？

✅ **核心回答**：
CIReVL 是两阶段方法（captioning + LLM 重写），本文基线 OSrCIR 改为单阶段反思式推理。本文在 OSrCIR 之上进一步引入代理图闭环，形成**三阶段迭代**。

### F3 为什么没和最新的 CIR 方法（如 2026 年论文）对比？

✅ **核心回答**：
OSrCIR 是 2025 CVPR Highlight，本毕业设计以其为基线已是前沿。2026 新方法多数仍未开源代码或不够稳定，且评估时期可能 benchmark 有更新。如有机会会在后续工作中对比。

### F4 和文生图 + 图图检索的思路相比？

✅ **核心回答**：
单纯"文生图 → 图图检索"会被 AI 幻觉带偏（我们也在早期实验中验证过），本方法的关键是**把生图作为辅助信号而非主信号**（α=0.9）+ 用反幻觉 prompt 让生图帮助 MLLM 自校正。

## 类别 G：总结性 & 自我评价

### G1 ⭐ 这项工作的最大贡献是什么？

✅ **核心回答**：
**首次系统性论证了"AI 生成图像可以作为 CIR 检索的可用辅助信号"**，并给出控制其噪声的具体机制（反幻觉 prompt + 保守融合权重）。

### G2 ⭐ 这项工作最大的局限是什么？

✅ **核心回答**：
1. **依赖 MLLM + T2I API**：成本 + 速度 + 可复现性问题
2. **参数敏感**：GeneCIS 需要 per-subset 调参
3. **基线不是 GPT-4o**：绝对数值低于原论文
4. **仅零样本设定**：未在监督训练上验证

### G3 ⭐ 如果重做一遍，你会如何改进？

✅ **核心回答**：
- 引入**多代理图 + CLIP 质量筛选**（生成 3 张，选最接近语义的 1 张）
- 训练一个**轻量 α/β 预测网络**（几万参数），从 query 特征预测最优参数
- 在**监督 CIR 数据**上验证 backbone 可迁移性
