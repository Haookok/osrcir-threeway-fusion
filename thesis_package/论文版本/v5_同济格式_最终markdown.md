## 摘 要

组合式图像检索根据一张参考图像与一段修改文本检索目标图像。在零样本设定下，代表方法 OSrCIR 借助多模态大语言模型直接推理目标描述再经 CLIP 检索，流程无需训练，但整体局限于文本空间，缺乏对描述质量的视觉验证，一旦初始描述偏离修改意图便难以纠正。

本文提出一种三路融合框架。视觉代理机制将初始描述经文生图模型还原为代理图像，为检索提供图像空间信号；反幻觉提示词策略将代理图严格限定为诊断工具，防止生成图像中的虚构细节流入目标描述；在 CLIP 特征空间对原始描述、精炼描述与代理图特征加权融合，使三路信号互补，降低单路失误导致的退化风险。针对 GeneCIS 这类短文本、小图库任务，进一步配套专用提示词与任务自适应参数。

在 FashionIQ、CIRCO、CIRR 与 GeneCIS 共 9 个基准的全量实验中，所提方法对全部 35 项评估指标均取得正向提升，无持平、无退化；9 个主指标的相对提升范围为 +5.2% ~ +84.5%，全量指标相对提升中位数为 +26%，验证了视觉代理与描述融合在零样本组合式图像检索中的有效性与通用性。

**关键词：** 零样本组合式图像检索；视觉代理；反幻觉提示词；描述融合；多模态大语言模型

## Abstract

Composed Image Retrieval (CIR) retrieves target images given a reference image and a modification text. In the zero-shot setting, the representative method OSrCIR directly infers a target description with a multimodal large language model and performs retrieval with CLIP. The pipeline requires no training, yet operates entirely in the text space and lacks a visual verification mechanism, so an inaccurate initial description can hardly be recovered.

This thesis proposes a Three-Way Fusion framework. A visual proxy mechanism converts the initial description into a proxy image via a text-to-image model, providing an image-space retrieval signal. An anti-hallucination prompting strategy restricts the proxy image to a diagnostic role, preventing hallucinated details from the generated image from leaking into the target description. The CLIP features of the original description, the refined description, and the proxy image are then fused with weights so that the three signals complement each other and a single failing path does not drag down overall performance. For GeneCIS-style tasks with short texts and small galleries, dedicated prompts and task-adaptive parameters are further adopted.

On nine benchmarks across FashionIQ, CIRCO, CIRR and GeneCIS, the proposed method improves all 35 evaluation metrics with neither ties nor regressions. The nine major metrics gain +5.2% to +84.5% relatively, with a median relative improvement of +26% over all 35 metrics, confirming the effectiveness and generality of the proposed framework.

**Keywords:** Zero-Shot Composed Image Retrieval; Visual Proxy; Anti-Hallucination Prompting; Description Fusion; Multimodal Large Language Model

## 第 1 章 绪论

### 1.1 研究背景与意义

图像检索是计算机视觉领域的经典问题，其目标是根据给定查询在大规模图库中找到语义最相关的图像 [[1](#ref1)]。传统图像检索通常按查询模态被划分为两类：以图搜图（content-based image retrieval）和以文搜图（text-based image retrieval）。前者依赖低层视觉特征或深度特征的相似度匹配，后者依赖文本—图像之间的跨模态对齐。随着视觉内容的爆炸式增长与用户表达习惯的日益丰富，单一模态的查询方式已经难以覆盖实际需求——用户越来越多地希望用"自然图像 + 自然语言修饰"的方式定位目标图像。

一个典型场景是电商平台的商品检索：用户上传一张心仪的连衣裙图片，同时希望系统返回"颜色改为深蓝、袖子加长"的同款款式；另一个常见场景是图像创作辅助工具中的素材替换，用户希望找到"保留当前构图、但把背景中的城市换成山林"的图片。这类查询的共同特征是**既包含视觉参考、又包含语义修改**，二者之间不是简单叠加，而是要求系统能够**有选择地保留与修改**不同属性。为应对这类需求，组合式图像检索（Composed Image Retrieval, CIR）任务被正式提出并形式化 [[2](#ref2), [3](#ref3)]。

组合式图像检索的形式化输入包括一张参考图像和一段自然语言修改文本，系统需要在候选图库中返回既保留参考图关键内容、又正确应用文本修改的目标图像。与传统检索相比，该任务的难度不仅来自跨模态理解本身，更来自"组合语义"的精细划分——哪些属性应被保留、哪些应被修改、修改的方向和程度如何界定，这些都需要模型在推理过程中动态判断。在学术界，FashionIQ [[18](#ref18)]、CIRR [[19](#ref19)]、CIRCO [[7](#ref7)]、GeneCIS [[20](#ref20)] 等基准先后被提出，分别覆盖服饰细粒度修改、自然场景语义替换、开放域组合查询和条件相似度判断等不同任务形态，逐步形成了较为完整的评测体系。

在众多研究方向中，**零样本组合式图像检索（Zero-Shot CIR, ZS-CIR）** 近年受到特别关注。其设定是：模型不能在目标数据集上做任何监督训练，必须依赖预训练视觉语言模型完成推理与匹配。这一设定之所以受到重视，既有学术原因也有工程原因——学术上，零样本设定消除了对大规模三元组标注的依赖，使方法的泛化能力成为主要评价维度；工程上，真实产品场景（不同电商、不同垂直领域）通常难以提供足够的监督数据，零样本方法更贴近实际可落地的技术路线。CLIP [[6](#ref6)] 及其衍生模型为 ZS-CIR 提供了基础跨模态表征能力，而 GPT-4o [[13](#ref13)]、Qwen-VL [[14](#ref14)]、LLaVA [[15](#ref15)] 等多模态大语言模型（MLLM）则进一步提供了组合语义推理能力，使"用语言直接描述目标图像、再用 CLIP 检索"这一简洁路径具备了工程可行性。

沿着这一思路，CVPR 2025 Highlight 工作 OSrCIR [[11](#ref11)] 通过反思式思维链（Reflective Chain-of-Thought）组织单阶段推理，成为近年 ZS-CIR 方向的代表方法之一。然而，OSrCIR 乃至同类基于推理的方法都存在一个结构性缺陷——**整个推理过程从参考图到最终检索完全停留在文本路径上**。MLLM 输出目标描述后，这段描述会被 CLIP 文本编码器直接消费用于检索，系统从未在图像空间对该描述做过任何验证。一旦初始描述对颜色、物体类别或局部属性的理解出现偏差，整条管线就会沿着错误方向持续检索，而没有机会从其它路径纠正。本文在 FashionIQ dress 200 样本上的抽样检查显示，Qwen-VL-Max 基线的初始描述在属性级的误判率合计约 18%，这些误判直接反映为检索指标下降。

本文围绕上述结构性缺陷展开研究，核心思路是**为纯文本管线补充一条可验证的图像空间信号**，并通过合理的特征融合让多路信号互相补充。相关研究不仅对组合式图像检索任务本身有直接价值，也为"如何把 AI 生成图像安全地用作推理中间产物"这一更一般性问题提供了方法论参考，在电商搜索、图像编辑辅助、内容创作检索等实际应用中具有潜在落地空间。

### 1.2 国内外研究现状

从现有文献来看，零样本组合式图像检索方法大体可以分为两类。

第一类是基于映射的方法。此类方法的基本思想是将参考图像映射到文本空间，再与修改文本组合形成查询表示。例如，Pic2Word [[5](#ref5)] 将图像表示转换为可被文本编码器处理的伪词向量，从而在 CLIP 文本空间中表达图像内容；SEARLE [[4](#ref4)] 和 iSEARLE [[8](#ref8)] 通过文本反转等技术进一步增强了这种映射方式；LinCIR [[9](#ref9)] 则尝试只利用文本数据训练轻量映射模块。该路线的优点是推理流程较简单、效率较高，但其局限在于图像被映射到文本空间时容易丢失细粒度视觉信息。

第二类是基于推理的方法。该类方法利用多模态大语言模型直接理解参考图像和修改文本，并生成目标图像描述。CIReVL [[10](#ref10)] 采用"先描述参考图，再根据修改文本推理目标图"的两阶段策略，而 OSrCIR [[11](#ref11)] 则进一步提出单阶段反思推理，使模型在一个统一流程中完成理解、分析和生成。OSrCIR 利用反思式思维链（Reflective Chain-of-Thought）组织推理步骤，在多个标准数据集上取得了优于已有无训练方法的表现，成为该方向的重要代表工作。

尽管如此，现有基于推理的方法仍存在明显不足。一方面，模型主要输出一条文本路径，后续检索几乎完全依赖这条路径是否准确；另一方面，模型生成描述之后缺少可回看的视觉中间结果，因此很难判断推理是否偏离目标。换言之，当前方法在"推理能力"方面已经较强，但在"验证能力"和"融合能力"方面仍有较大改进空间。

### 1.3 现有方法存在的问题

通过对基线方法复现和多轮实验分析，本文将现有方法的问题总结为以下三点：

1. **初始描述缺乏视觉验证**。基线方法通常只调用一次多模态大语言模型，直接生成目标描述。如果该描述对颜色、物体或局部属性的理解出现误差，后续 CLIP 检索便会受到持续影响。
2. **单一路径检索较为脆弱**。CLIP 对文本表达方式较为敏感，即使两个描述语义接近，在特征空间中的位置也可能不同。检索系统完全依赖一条文本描述时，整体性能受单点误差影响较大。
3. **AI 生成图像虽然有潜在价值，但容易引入幻觉**。若直接让模型根据代理图重新写描述，模型往往会把代理图中的背景、材质、环境等虚构信息带入最终检索描述，反而使检索偏离目标。

因此，本文的关键问题在于：如何将 AI 生成图像作为一种"可用但不可信"的辅助信号纳入检索流程，同时避免其成为新的误差来源。

### 1.4 本文主要工作

针对上述问题，本文围绕 OSrCIR 基线方法提出了三路融合改进框架。主要工作如下：

1. **提出视觉代理机制**。将第一轮生成的目标描述输入文生图模型，生成代理图像，为检索阶段提供图像空间的辅助信号。
2. **设计反幻觉精炼策略**。通过专门设计的提示词，要求模型将代理图仅作为诊断工具使用，只检查修改是否正确应用，而不能将代理图中的细节直接写入目标描述。
3. **提出描述融合与三路融合方案**。在 CLIP 特征空间中对原始描述、精炼描述及代理图特征进行加权融合，兼顾稳健性与检索精度。
4. **针对 GeneCIS 等小图库短文本任务**，进一步探索专用提示词与任务自适应参数设置，验证方法在不同任务类型上的可扩展性。

## 第 2 章 相关工作与技术基础

### 2.1 组合式图像检索任务定义

组合式图像检索以一张参考图像和一段自然语言修改文本作为查询，要求系统从候选图库中返回既保留参考图关键内容、又正确应用文本修改的目标图像。形式化地，设参考图像为 $I_r$，修改文本为 $t$，候选图库为 $\mathcal{G}=\{I_1,I_2,\ldots,I_N\}$，检索系统需要在 $\mathcal{G}$ 中找到最接近"$I_r$ 经 $t$ 修改后"的图像 $I_t$。

与传统的基于单一模态的检索相比，组合式图像检索的输入天然跨越了视觉和语言两个模态，且这两类输入不是相加关系而是交互关系：文本并非对图像的整体重述，而是对图像中特定属性的局部修正。例如 FashionIQ [[18](#ref18)] 数据集中，参考图像为"红色短袖连衣裙"，修改文本为"change the color to blue and make it longer"，目标图像则是"蓝色长款连衣裙"，模型需要同时保留"连衣裙"这一类别与款式特征、替换颜色属性、修改长度属性。CIRCO [[7](#ref7)] 和 CIRR [[19](#ref19)] 进一步把场景从服饰扩展到开放域，修改文本常常涉及物体替换、场景变化等更复杂的语义操作；GeneCIS [[20](#ref20)] 则走向另一种极端，其修改文本被压缩为一两个单词，每个查询只对应一个约 14 张图的微型图库，侧重考察模型对细粒度条件相似度的判断能力。

按是否允许在目标数据集上进行监督训练，该任务可划分为全监督、弱监督和零样本三类。全监督方法依赖大规模标注三元组 $(I_r, t, I_t)$，在对应数据集上表现良好但迁移能力有限；零样本方法则要求系统在**不使用任何目标数据集标注**的前提下完成检索。零样本设定更贴近实际产品化需求——不同电商、不同搜索场景通常难以提供大规模三元组监督，因此基于通用预训练模型的零样本方案具备更强的工程价值。本文研究即属于零样本设定。

### 2.2 基于映射的方法

基于映射的方法延续了 CLIP 图文对齐的范式，其核心思想是**把参考图像翻译成"伪文本"**，再与修改文本在文本空间中拼接，最后用 CLIP 文本编码器得到查询表示。该路线由 Pic2Word [[5](#ref5)] 提出：训练一个轻量映射网络 $\phi(\cdot)$，把 CLIP 图像特征映射为一个"伪词向量"$v^\*$，该向量可以被 CLIP 文本编码器像真实 token 一样接受，从而允许将参考图表示嵌入到 "a photo of $v^\*$, but longer and in blue" 这类 prompt 模板中。训练过程只需要无标注的图像，通过对比 $\phi(\text{CLIP}_I(I))$ 与真实文本 caption 的编码来约束映射模块。

SEARLE [[4](#ref4)] 沿用了"图像到伪词"的思路，但把 Pic2Word 使用的对比学习替换为文本反转（textual inversion）：利用 CLIP 的可逆性，通过优化使伪词在嵌入空间中的表示能最大限度复现图像的 CLIP 特征。iSEARLE [[8](#ref8)] 在此基础上进一步改善了训练稳定性和可迁移性。LinCIR [[9](#ref9)] 把训练成本推向极限——仅用文本数据训练映射模块，推理时再把图像特征按同一变换送入管线，避免了对大规模图像的依赖。

该路线的共同优点是结构清晰、推理代价低、与 CLIP 的耦合度高，在 FashionIQ 等细粒度服饰数据集上取得了不错的结果。其瓶颈也相对明确：图像被强制压缩为文本空间中的一个向量（甚至一个 token）后，颜色、材质、纹理、布局这些细粒度视觉信号会显著损失。当修改文本要求保持图像中"纹理一致但颜色改变"时，映射后的伪词已无法分辨纹理与颜色，系统只能依靠修改文本的语义指引完成检索，真实的视觉证据已经在管线的前端被丢弃。

### 2.3 基于推理的方法

与映射方法的"压缩—拼接"思路相反，基于推理的方法选择**直接调用多模态大语言模型完成组合式理解**，把参考图像和修改文本都交给模型，让其用自然语言输出目标图像描述，再把描述送入 CLIP 文本编码器。这一路线的起点是 CIReVL [[10](#ref10)]：第一步用 BLIP/LLaVA 等视觉语言模型为参考图写出详细 caption，第二步用纯语言模型根据 caption 和修改文本推导出目标 caption。两步分离的设计便于调试，但在第一步生成的 caption 就会损失大量图像细节，后续第二步只能在残缺的文本输入上继续推理。

OSrCIR [[11](#ref11)] 是目前最具代表性的单阶段方法，也是本文的基线。该工作将 CIReVL 的两阶段压缩为一次调用——参考图和修改文本被同时送入 MLLM，并在 prompt 中要求模型按"反思式思维链"（Reflective Chain-of-Thought）逐步输出：先观察参考图内容、再分析修改文本意图、再推理目标图像应有的属性、最后给出目标描述。相比两阶段方案，OSrCIR 在单次调用中即可完成语义交互，避免了 caption 中间表示带来的信息瓶颈，并在 FashionIQ、CIRR、CIRCO、GeneCIS 等主流基准上取得了当时最好的零样本成绩。

尽管 OSrCIR 证明了"让模型直接推理"这一路径的潜力，它仍然存在几个层面的遗留问题。其一，**整个流程只在文本空间完成**：输入虽然包含图像，但输出仅为一段文字描述，系统从未把这段描述在图像空间中"验证"过一次——模型是否真的理解了"把裙子加长"这一修改，在管线中没有任何图像层面的证据。其二，**单路径依赖**：一旦 MLLM 在某个样本上输出了错误描述（比如漏掉颜色变化、把物体替换方向搞反），CLIP 会严格按照该错误描述进行检索，整条管线没有兜底机制。其三，**对 MLLM 的能力高度敏感**：OSrCIR 原论文使用 GPT-4o [[13](#ref13)]，本文因接口原因替换为 Qwen-VL-Max [[14](#ref14)]，仅这一处差别就在 FashionIQ dress 上造成了 R@10 从 29.70 到 15.80 的绝对下降（见 4.1.3 节表 4-1），说明文本推理路径对上游模型性能高度脆弱。本文的工作正是在承认该路径脆弱性的前提下，尝试通过图像空间的辅助路径对其进行补强。

### 2.4 技术基础

#### 2.4.1 CLIP

CLIP [[6](#ref6)] 由 OpenAI 在 2021 年提出，通过在 4 亿图文对上进行对比学习，训练出一个将图像和文本编码到同一向量空间的双塔模型。其训练目标是让匹配的图文对在特征空间中距离更近、不匹配的更远，形式上等价于对称的 InfoNCE 损失。推理时，图像编码器 $E_I$ 和文本编码器 $E_T$ 可独立使用，任何一段文本和任何一张图像都可以被映射到同一 $d$ 维球面，并通过余弦相似度 $\mathrm{sim}(a,b)=\frac{a\cdot b}{\|a\|\|b\|}$ 计算相关程度。

本文与 OSrCIR 原论文保持一致，采用 CLIP ViT-L/14 作为特征编码器。该模型的图像分支基于 Vision Transformer，输入 224×224 图像被切分为 14×14 的 256 个 patch，每个 patch 经线性投影后加上可学习位置编码，送入 24 层 Transformer Encoder，最终取 [CLS] 位置的输出作为图像特征；文本分支使用 BPE 分词后的 77 个 token 作为最大上下文，经 12 层 Transformer 后取 [EOT] 位置作为文本特征。图像与文本特征维度均为 768，经对比学习已在语义层面高度对齐。

选择 CLIP ViT-L/14 的另一考虑是其对后续实验的计算代价可控。本文全量实验中 CIRCO 的候选图库达 123K，FashionIQ 三个子集加起来约 50K，GeneCIS 每个查询的图库仅 14 张。CLIP 特征一旦离线提取并缓存即可反复使用，最终实际运行中所有候选图库的 CLIP 特征都事先缓存到 `precomputed_cache/precomputed/` 目录，检索阶段仅需计算查询端特征，大幅降低了重复实验成本。

#### 2.4.2 多模态大语言模型

多模态大语言模型通过将视觉编码器的输出投影到语言模型的 token 空间，使大语言模型能够在同一上下文中消化图像和文本输入，并按生成式范式输出自然语言。近两年 GPT-4o [[13](#ref13)]、Qwen-VL [[14](#ref14)]、LLaVA [[15](#ref15)] 等工作显著提升了这一方向的可用性，尤其在图像细节描述、视觉推理和多轮视觉对话方面已具备实用能力。

本文实验使用的 Qwen-VL-Max 是阿里云 DashScope 平台提供的闭源 MLLM，支持图像和文本混合输入，单次调用最长可处理约 30K token，足以容纳 OSrCIR 所需的反思式思维链 prompt 和参考图像输入。在提示词工程层面，该模型对系统消息、用户消息、few-shot 样例等结构有良好响应，能够按 prompt 指令逐步输出中间推理过程与最终目标描述。本文所有涉及 MLLM 的调用（包括基线复现、代理图生成的初始描述、反幻觉 prompt 下的第二轮精炼）均统一通过该模型完成，单次调用耗时约 4–7 秒，单条描述生成平均费用约 0.03 元人民币，整个项目累计 MLLM 开销约 90 元。

反思式思维链是本文继承自 OSrCIR 的 prompt 组织方式。其基本结构是要求模型依次回答四个子问题："参考图像中有什么？修改文本要求做什么变化？修改后图像应该是什么样？输出目标描述。"该结构的关键在于让模型把隐式的视觉—语言推理拆成显式步骤，避免直接要求"写出目标描述"导致的跳跃式输出。本文的 V7 反幻觉 prompt 继承了这一骨架（详见 3.3.2 节），但在最后一步增加了对幻觉的显式约束。

#### 2.4.3 文生图模型

文生图模型以文本作为条件、图像为输出，近年以扩散模型为主流范式。DALL·E 2 [[16](#ref16)]、Stable Diffusion [[17](#ref17)] 以及各类商业 API（MidJourney、MiniMax image-01 等）均属于这一类。其核心机制是从纯噪声出发，通过若干步去噪操作逐步还原出与文本条件一致的图像，文本条件通常由 CLIP 文本编码器或专用 T5 编码器提供。

本文使用 MiniMax image-01 生成代理图，主要出于两点考虑：其一，该接口稳定，单次调用耗时约 8–12 秒，成本约 0.03 元，适合大规模批量生成；其二，其文本理解能力足以处理 MLLM 输出的长描述（通常 80–150 个英文词），不会因输入过长而频繁截断。需要强调的是，本文并不把代理图作为最终检索结果，而是把它当作**管线的中间产物**：代理图承载了"MLLM 对目标图像的想象"，在后续精炼和检索阶段被同时用于诊断和特征补充。代理图的幻觉问题（虚构背景、材质、环境等）并非本文回避的对象，相反，3.3 节提出的反幻觉 prompt 正是围绕如何"使用但不轻信"代理图这一矛盾展开设计。

### 2.5 本章小结

本章首先形式化了组合式图像检索任务的输入输出与零样本设定，随后系统梳理了基于映射与基于推理两类代表方法的动机、机制和瓶颈——前者以牺牲视觉细节换取管线简洁性，后者虽然保留了视觉输入，但最终输出仍被限制在纯文本路径上，缺乏图像层面的验证。针对这一结构性缺陷，本章最后介绍了本文三路融合方法所依赖的三类基础模型——CLIP、多模态大语言模型、文生图模型——的原理、实现细节以及本文的具体选型。以上内容为第 3 章方法设计提供了概念与技术前提。

## 第 3 章 三路融合方法

### 3.1 方法总体框架

本文提出的三路融合方法在 OSrCIR 基线的"MLLM 推理 + CLIP 检索"两段式管线之上，插入了**视觉代理生成**和**反幻觉描述精炼**两个新环节，并把最终检索得分建立在三条互补路径的特征融合之上。整体流程如图 [3-1](#fig-pipeline) 所示：

1. 输入参考图像 $I_r$ 与修改文本 $t$，由多模态大语言模型按反思式思维链生成第一轮目标描述 $D_1$。
2. 将 $D_1$ 送入文生图模型，生成代理图像 $P$。
3. 将 $I_r$、$P$ 和 $t$ 同时送入 MLLM，通过 V7 反幻觉提示词生成第二轮精炼描述 $D_2$。
4. 分别用 CLIP 对 $D_1$、$D_2$ 和 $P$ 编码，在特征层面做文本融合得到 $\mathbf{f}_{\text{text}}$，再与图像特征 $\mathrm{CLIP}(P)$ 在得分层面做三路融合，得到最终检索排序。

**图 3-1** 三路融合方法总体流程。第一阶段生成初始描述 $D_1$ 并通过文生图模型生成代理图像；第二阶段利用反幻觉提示词精炼描述；检索阶段融合三路 CLIP 特征计算最终得分。

该框架有三处与基线方法的本质区别。第一，**引入了图像空间信号**：基线的输出始终是文本，而本方法在管线中间产生了一张代理图像，并把它作为独立的一路检索信号，而不只是 MLLM 推理的中间变量。第二，**对第二轮精炼的角色重新定义**：精炼的任务不是"重写描述"，而是"用代理图核对描述是否与修改意图一致"；这一角色限制通过 V7 反幻觉 prompt 显式施加，避免代理图的幻觉细节污染最终描述。第三，**最终得分是融合而非替换**：$D_1$、$D_2$、$P$ 三路各自独立计算与候选图的相似度，再按可调权重合并，任一路径的失误都可以被另外两路的证据部分抵消，从而降低单路脆弱性带来的退化风险。

### 3.2 视觉代理机制

#### 3.2.1 设计动机

OSrCIR 基线最核心的缺陷是**整条管线从未在图像空间做过任何验证**——MLLM 输出的 $D_1$ 直接被 CLIP 文本编码器消费，系统完全没有机会判断这段描述是否真的符合修改意图。项目早期在 FashionIQ dress 200 样本上抽样检查了基线输出，发现 MLLM 在颜色、长度、袖型等属性上的误判率合计约 18%，典型错误包括把"navy blue"写成"dark green"、把"sleeveless"漏写为普通款式、把"longer"的修改意图错理解为"shorter"等。这些错误在纯文本路径上无从察觉，只能通过事后看检索结果倒推。

视觉代理机制的基本设想是："既然 MLLM 已经输出了它对目标图像的理解，那就让这个理解在图像空间中显形一次"。具体做法是把 $D_1$ 直接送入文生图模型得到代理图 $P$，其承载的信息等价于"如果 MLLM 是对的，目标图像应该长这样"。之后无论是用于第二轮描述的对照、还是作为一路独立的检索信号，代理图都提供了一个**此前完全缺失的视觉锚点**，使系统有机会在图像层面对文本推理进行二次确认。

#### 3.2.2 信息路径与预期作用

从信息流角度看，视觉代理把原本的单向文本管线 "$(I_r, t) \to D_1 \to \text{检索}$" 改造为一条闭环 "$(I_r, t) \to D_1 \to P \to \text{对 } D_1 \text{ 的验证 } \to D_2$"。$P$ 在这条回路中同时扮演两种角色：

1. **诊断信号**（供第二轮精炼使用）：$P$ 并非完美的目标图像，但它是"按 $D_1$ 字面理解得到的图像"。如果 $P$ 中缺失了某个修改意图（例如 $D_1$ 漏写了颜色变化，$P$ 就会保留参考图的颜色），模型在第二轮精炼时就能发现差异并进行纠正。
2. **检索信号**（供最终得分融合使用）：$P$ 的 CLIP 特征直接参与与候选图库的相似度计算。由于 $P$ 自身即为图像，它与候选图的比较发生在 CLIP 图像空间内部，绕开了文本到图像的跨模态映射瓶颈。对于那些 MLLM 难以准确表达但文生图模型能够"画出来"的视觉特征（例如特定的布料纹理、光照氛围），代理图路径能提供文本路径无法替代的补充。

在 FashionIQ dress 200 样本的早期试验中，仅把 CLIP($P$) 作为一路辅助信号参与检索（Plan A 后融合方案，$\alpha=0.9$），已能将 R@10 从基线的 15.80 提升到 17.40（+1.6pp，相对提升 +10.1%）。这一正向结果证实：代理图尽管存在生成噪声，但其承载的视觉信息与 CLIP 文本路径存在信息差，合并后能带来真实的增益。

#### 3.2.3 生成方式与成本

代理图由 MiniMax image-01 生成，输入即为 $D_1$ 本身，不再附加任何引导词或样式修饰，以保证 $P$ 最大限度反映 $D_1$ 的字面内容。单次生成耗时约 8–12 秒，费用约 0.03 元人民币，输出尺寸固定为 512×512。对于 FashionIQ dress 全量 1903 个查询，整个代理图生成阶段耗时约 5 小时、费用约 57 元。所有代理图按查询 ID 缓存到 `proxy_cache/` 目录（PNG 格式，平均 180KB），在后续消融和参数搜索中反复使用，不再重复调用 API。

### 3.3 反幻觉描述精炼

#### 3.3.1 直接使用代理图的失败尝试

视觉代理机制的第一个直觉用法是："既然有了代理图，就让 MLLM 看着代理图重新写描述"。项目中按这一思路设计了 V1–V6 共 6 个版本的精炼 prompt，在 CIRR 200 样本上进行迭代测试（详见 3.3.3 节图 [3-2](#fig-prompt)），结果全部出现不同程度的退化，V6 甚至出现灾难性下降（R@10 从 67.0 跌至 49.5，相对下降 26%）。

通过对 MLLM 输出的逐条检查，项目确认了退化的根本原因是**代理图的幻觉细节被模型写入了最终描述**。例如在 CIRR 的一个样本中，修改文本只要求"把狗换成猫"，参考图是一只狗站在草地上；代理图由于文生图模型的自行脑补，在画面左上角添加了一只不存在的蝴蝶和远处的栅栏。V2 版精炼 prompt 让模型"综合参考图和代理图写出目标描述"，结果输出变成"a cat standing on grass, with a butterfly in the background and a wooden fence behind"——检索时 CLIP 会按"猫 + 蝴蝶 + 栅栏"寻找候选图，真正的目标图（只有猫和草地）反而被排到后面。这类失败在 V1–V6 中反复出现：背景、道具、光照、材质等本应从参考图继承的视觉属性，被代理图的虚构版本替换。

本质上，**代理图的"可信性"和"可用性"之间存在根本矛盾**——它对诊断修改是否应用有价值，但对具体视觉细节并不可信。任何把两者合二为一的 prompt 设计都会失败。这一观察直接推导出 V7 的核心原则。

#### 3.3.2 V7 反幻觉策略

V7 提示词的根本改动是**在任务层面把代理图从"描述来源"降级为"诊断工具"**，使模型在生成 $D_2$ 时不再有理由从 $P$ 中复制细节。具体通过以下几条约束实现：

1. **显式声明代理图性质**。prompt 开头明确说明："the proxy image is AI-generated from the initial description, it may contain hallucinated details such as wrong background, fabricated objects, inaccurate colors"，先打破模型默认把所有输入视作可信的倾向。
2. **限定代理图的使用方式**。要求模型**仅使用代理图回答一个问题**——"Is the modification correctly reflected in the proxy?"（修改意图是否正确体现）。如果是，保留 $D_1$ 的原有描述；如果否，在 $D_1$ 基础上做**最小修正**。
3. **禁止属性复制**。prompt 显式列出禁止从代理图复制的属性类别：background, environment, lighting, fabric texture, secondary objects not mentioned in the modification。这一黑名单在项目中通过逐版本失败案例归纳而成。
4. **锁定描述长度**。要求 $D_2$ 长度不得超过 $D_1$ 的 1.2 倍，防止"重写描述"式的扩张。
5. **保留反思式思维链**。V7 继承 OSrCIR 的四步结构（观察参考图→分析修改→推理目标→输出描述），但在第三步增加"对照代理图检查"这一子步骤，使反思过程覆盖视觉验证环节。

以 CIRR 的一个典型样本为例："change the dog to a cat"——$D_1$ 生成结果为"a white cat sitting on grass in a park"，代理图中额外出现了一条小路和远处行人；V7 下的 $D_2$ 最终为"a cat sitting on grass"，只对 $D_1$ 做了极小的收紧修改，完全没有引入代理图中的虚构元素。基线 $D_1$ 直接检索时 Rank=8，$D_2$ 替换基线时 Rank=12（更差），V7 下融合检索时 Rank=2，显示了"诊断式使用"的价值。

#### 3.3.3 Prompt 演进过程

V1 到 V7 的迭代时间跨度约 5 周，每个版本均在 CIRR 200 样本上做 A/B 测试，累计 MLLM 调用约 1.2K 次，累计花费约 36 元。图 [3-2](#fig-prompt) 展示了七个版本在 R@10 上的变化轨迹：

**图 3-2** 不同版本 prompt 在 CIRR 数据集上的 R@10 表现（200 样本）。V6 因信息过载导致灾难性退化，V7 通过反幻觉策略实现突破。

- **V1**（参考图 + 代理图直接对比）：R@10 = 66.5，仅略优于基线 67.0 的持平水平，退化轻微但未证明代理图有效。
- **V2**（加入简单思维链）：R@10 = 64.0，思维链让模型从两张图都复制细节，负面效应放大。
- **V3**（去掉参考图，仅用代理图重写）：R@10 = 58.0，彻底丢失参考图信息，退化最严重。
- **V4**（恢复参考图 + CoT + 首次尝试强调代理图不可信）：R@10 = 66.0，有所回升但尚不稳定。
- **V5**（增加 Few-shot 示例）：R@10 = 65.5，示例反而让模型模仿示例的写法、忽略了当前样本的上下文。
- **V6**（将 $D_1$ 原文、思维链过程和代理图一并输入）：R@10 = 49.5，信息过载导致模型彻底混乱。
- **V7**（反幻觉策略 + 诊断式约束 + 长度锁）：R@10 = 67.5，**首次突破基线**。

这一迭代过程呈现出清晰的失败模式——所有"信息越多越好"的直觉尝试都导致退化，只有 V7 "明确减法"的设计方向获得了正向结果。该结论也与 MLLM 的一般使用经验相符：对于视觉生成类输入，应通过显式约束限制模型的自由度，而不是依赖其自行筛选。

### 3.4 描述融合策略

#### 3.4.1 单路替换的局限性

即使 V7 解决了幻觉问题，用 $D_2$ 替换 $D_1$ 的朴素方案仍然存在两个问题。其一，**非所有样本都需要精炼**——项目统计表明，在 CIRR 200 样本中约 62% 的 $D_1$ 本身已经是较好的描述，V7 精炼结果与 $D_1$ 几乎一致；这部分样本用 $D_2$ 替换相当于做了一次无效的 MLLM 调用，不带来提升也无损失。其二，**剩余 38% 的样本中，精炼并不总是成功**——仍有少量样本精炼后描述过于简短、丢失关键修饰词，反而使检索质量下降。基线 $D_1$ 上 FashionIQ dress R@10=15.80，直接替换为 $D_2$ 后 R@10 = 15.50（-0.30），消失的增益正是被这些退化样本抵消掉了。

**融合而非替换**的思路正是为处理上述分布不均问题。融合操作不依赖"样本是否需要精炼"的二值判断，而通过权重让每个样本自然获得两路的组合——对于 $D_1$ 已经足够好的样本，$D_2$ 与 $D_1$ 的 CLIP 特征高度接近，融合几乎等价于 $D_1$ 本身；对于精炼显著改进的样本，$D_2$ 带来的方向调整会通过融合传递到最终查询表示；对于精炼退化的样本，$D_1$ 的主导权重能把退化幅度控制在可接受范围。

#### 3.4.2 融合公式

设第一轮描述特征为 $\mathrm{CLIP}(D_1)$，第二轮精炼描述特征为 $\mathrm{CLIP}(D_2)$，文本融合特征定义为：

$$
\mathbf{f}_{\text{text}} = \mathrm{normalize}\left(\beta \cdot \mathrm{CLIP}(D_1) + (1-\beta) \cdot \mathrm{CLIP}(D_2)\right)
\tag{3-1}
$$

其中 $\beta \in [0,1]$ 控制对两路描述的信任程度。加权求和后必须进行 L2 归一化，这一步是必要的而不是美学选择：CLIP 的图文对齐是建立在单位球面上的余弦相似度之上，未归一化的加权向量会改变向量模长，使后续相似度得分同时受方向和模长两种因素影响，引入不可预期的偏差。归一化后，$\mathbf{f}_{\text{text}}$ 回到单位球面，检索得分的含义保持与原始 CLIP 一致。

本文默认取 $\beta = 0.7$，即以原始描述为主、精炼描述为辅。该选择的依据是 FashionIQ dress 200 样本上 $\beta \in \{0.3, 0.5, 0.7, 0.9, 1.0\}$ 的网格扫描结果——$\beta = 0.7$ 在 R@10 上取得最佳（18.30），$\beta = 0.5$ 退化到 17.60（精炼权重过大），$\beta = 1.0$ 等价于不用精炼（17.40），$\beta = 0.9$ 处于中间位置（17.95）。直观上，$\beta = 0.7$ 相当于"主要相信 $D_1$，但给 $D_2$ 留出足够的纠偏空间"。需要注意的是，最优 $\beta$ 并非对所有任务通用——GeneCIS 上的网格搜索表明不同子任务的最优 $\beta$ 差异很大（详见 4.3.2 节和表 [4-7](#tab-grid)），这也是本文对 GeneCIS 采用任务自适应参数的出发点。

### 3.5 三路融合检索公式

在完成文本特征融合后，将代理图像特征作为第三路并入最终得分。设候选图库图像为 $I_c$，代理图像特征为 $\mathrm{CLIP}(P)$，最终检索得分定义为：

$$
\mathrm{score}(I_c) = \alpha \cdot \mathrm{sim}\left(\mathbf{f}_{\text{text}}, \mathrm{CLIP}(I_c)\right) + (1-\alpha) \cdot \mathrm{sim}\left(\mathrm{CLIP}(P), \mathrm{CLIP}(I_c)\right)
\tag{3-2}
$$

其中 $\alpha \in [0,1]$ 控制文本路径与图像路径的权重。本文默认取 $\alpha = 0.9$，文本为主、图像为辅。

从几何角度理解，公式 (3-2) 相当于在 CLIP 单位球面上同时维护两种不同性质的查询向量——$\mathbf{f}_{\text{text}}$ 来自跨模态映射后的"文本位点"，$\mathrm{CLIP}(P)$ 来自直接的"图像位点"——并把二者对候选图的相似度以凸组合方式合并。当一条路径在某个样本上因推理偏差落到错误的语义区域时，另一条路径仍可能停留在正确区域附近，融合得分会把候选图的排序推向两条路径的**一致正确方向**。这也是本文将该框架称为"融合"而非"替换"的原因：三路之间是证据合并的关系，而不是互相否决的关系。

$\alpha = 0.9$ 的默认值基于两点考量。其一，文本路径的信息密度更高——$\mathbf{f}_{\text{text}}$ 综合了两轮 MLLM 推理的结果，而代理图只经过了一次"描述→图像"的映射，中间还注入了文生图模型的噪声。其二，代理图的分布偏差客观存在——文生图模型倾向于生成干净、居中、单主体、明亮的图像，与真实数据集中的候选图存在领域差异，如果赋予过大权重会系统性地把检索引向"AI 风格"候选图。$\alpha = 0.9$ 在 FashionIQ dress 200 样本上的网格扫描中取得最佳 R@10（19.00），$\alpha = 1.0$（不用代理图）为 18.30，$\alpha = 0.7$ 降为 17.20。与 $\beta$ 类似，GeneCIS 上不同子任务的最优 $\alpha$ 存在差异，具体结果见 4.3.2 节。

### 3.6 方法特点总结

与 OSrCIR 基线相比，三路融合方法在以下几个维度上与其区别开来：

1. **保留无训练特性**。整个管线不涉及任何在目标数据集上的监督训练或微调，仅依赖 MLLM、CLIP 和文生图模型的预训练参数。从计算代价看，新增的视觉代理阶段相当于每个查询多一次文生图调用和一次 MLLM 精炼调用，推理时间约增加 15 秒、费用约 0.06 元，但不需要任何训练基础设施。
2. **引入图像空间信号**。代理图在管线中同时担任诊断和检索两种角色，使系统从纯文本管线升级为图文交叉管线。这一改动实际是在回答一个方法论问题：**如果 MLLM 的输出无法直接验证，能否通过引入新模型把它投射到可验证的空间？** 本文的答案是肯定的，代价是多一次文生图调用。
3. **通过融合增强稳健性**。$D_2$ 替换 $D_1$ 在 FashionIQ dress 上造成退化，而 $\beta = 0.7$ 的融合把同样的第二轮精炼信号转化为 +2.5pp 的增益。这一对比直接说明了"融合式使用 MLLM 第二轮输出"相比"替换式使用"在稳健性上的本质优势。
4. **参数化可调节性**。$\alpha$ 和 $\beta$ 把三路融合纳入一个两参数的统一框架，便于后续的消融、敏感性分析和任务自适应优化。4.3.2 节在 GeneCIS 上的参数搜索即建立在这一统一接口之上。

同时也需要明确本方法的几条边界。其一，改进的上限仍受 MLLM 自身能力约束——当 $D_1$ 完全错误时，代理图也会"忠实地画错"，本文框架无法从错误的 $D_1$ 中恢复出正确的目标。其二，文生图模型的领域偏差会被代理图路径带入检索，对于特定垂直领域（如特定品牌服饰）可能需要替换更合适的 T2I 模型。其三，参数 $\alpha$、$\beta$ 的最优值存在任务间差异，默认值并非对所有数据集最优，这一观察在 4.3.2 节有专门讨论。

### 3.7 本章小结

本章围绕"如何在不训练的前提下对 OSrCIR 基线做非平凡改进"这一核心问题，系统阐述了三路融合方法的设计思路和实现细节。第 3.1 节概述总体流程，第 3.2 节讨论视觉代理的动机与信息路径，第 3.3 节详细交代了反幻觉 prompt 从 V1 失败到 V7 成功的迭代过程，第 3.4 节论证了描述融合相较于单路替换的必要性，第 3.5 节给出三路融合的最终检索公式并解释了默认参数的几何与工程依据。本章内容为第 4 章的全量实验验证提供了完整的方法描述，也为后续消融分析和参数敏感性研究建立了统一的数学接口。

## 第 4 章 实验与结果分析

### 4.1 实验设置

#### 4.1.1 数据集

本文在 4 个标准基准数据集共 9 个子任务上进行评估，涵盖服饰细粒度检索、开放域检索、自然场景组合检索和条件相似度检索四种典型场景：

- **FashionIQ** [[18](#ref18)]：细粒度服饰检索数据集，包含 dress、shirt、toptee 三个子集，分别聚焦连衣裙、衬衫和上衣类。修改文本多为短语级指令（如 "is shorter and has stripes"），候选图库为对应子集的所有图像，dress 集约 1903 张、shirt 集约 2038 张、toptee 集约 1961 张。评估指标采用 R@10 和 R@50，与原论文保持一致。
- **CIRCO** [[7](#ref7)]：开放域组合检索数据集，查询共 220 个，候选图库为 COCO 2017 验证集全部约 123K 张图像。与 FashionIQ 不同，CIRCO 每个查询有多个正确目标（multi-ground-truth），评估指标为 mAP@5、mAP@10、mAP@25、mAP@50。由于库规模大、正确答案多样，CIRCO 被视为最具开放域挑战性的 ZS-CIR 基准。
- **CIRR** [[19](#ref19)]：自然场景组合检索数据集，共 4148 个查询、2297 张候选图，修改文本语义丰富且常涉及物体替换、位置关系变化。除全库检索指标 R@K 外，CIRR 还设计了子集检索指标 R_sub@K——查询时候选仅限于 6 张语义相关图像，用于度量对细微差别的区分能力。
- **GeneCIS** [[20](#ref20)]：条件相似度基准，由 change_object（1960 样本）、focus_object（1960 样本）、change_attribute（2110 样本）、focus_attribute（1998 样本）四个子任务组成，每个子任务共约 8K 条查询。与前三者不同，GeneCIS 的修改文本被压缩为一到两个英文单词（例如 "dog → cat" 或 "focus on: red"），每个查询对应的候选仅约 14 张图，评估指标为 R@1、R@2、R@3。这一设定考察模型在**短文本 + 小图库**极端条件下的细粒度判别能力。

9 个子任务合计覆盖 35 项评估指标，构成了本文方法的完整评估矩阵。

#### 4.1.2 实现细节

本文所有实验的配置如下：

- **多模态大语言模型**：Qwen-VL-Max（阿里云 DashScope API），用于生成 $D_1$ 和第二轮精炼 $D_2$。单次调用平均耗时 4–7 秒，费用约 0.03 元。
- **文生图模型**：MiniMax image-01，用于根据 $D_1$ 生成代理图 $P$。输出固定为 512×512 PNG，单次调用平均耗时 8–12 秒，费用约 0.03 元。
- **视觉语言编码器**：CLIP ViT-L/14（OpenAI 开源权重 `ViT-L-14.pt`），用于编码 $D_1$、$D_2$、$P$ 及候选图库，与 OSrCIR 原论文完全一致。
- **默认融合参数**：$\beta = 0.7$、$\alpha = 0.9$，仅 GeneCIS 使用任务自适应参数（详见 4.3.2 节）。
- **运行环境**：描述生成与代理图生成阶段运行在 Linux 服务器（16 核 CPU、3.6GB 内存 + 9GB swap、Python 3.11、openai 2.29.0），CLIP 编码和大规模检索运行在 Windows 笔记本（RTX 4060 Laptop 8GB GPU、32GB RAM），候选图库特征均事先离线提取并缓存。
- **复现与调试开销**：9 个数据集累计 MLLM 调用约 14K 次、文生图调用约 6K 次，API 总花费约 140 元人民币。所有代理图、精炼描述和 CLIP 特征均写入本地缓存，便于后续参数扫描复用。

#### 4.1.3 关于基线复现的说明

本工作存在一处需要特别澄清的复现差异：OSrCIR 原论文使用的 MLLM 是 OpenAI GPT-4o [[13](#ref13)]，而本文因 API 可用性限制改用 Qwen-VL-Max [[14](#ref14)]。由于 MLLM 本身是 OSrCIR 管线的核心组件，这一替换直接影响基线的绝对数值。具体对比见表 [4-1](#tab-baseline)：在 FashionIQ dress 上 R@10 从 29.70 跌至 15.80（绝对下降 13.90pp），shirt 从 33.17 跌至 26.00，toptee 从 36.92 跌至 23.09。造成差距的主要原因是 Qwen-VL-Max 对 FashionIQ 英文修改文本的细粒度理解能力弱于 GPT-4o，尤其在颜色、款式的混合修改上误判率更高。

需要强调的是，这一绝对数值差异**不影响本文对改进方法有效性的结论**。原因在于：本文所有改进方案（视觉代理、V7 反幻觉精炼、描述融合、三路融合）均建立在同一 Qwen-VL-Max 基线之上，Baseline 与三路融合使用的是完全相同的 MLLM 实例、相同的 prompt 骨架、相同的 CLIP 编码器、相同的候选图库特征；方法间的差异只来自本文新增的环节本身。因此比较 "Baseline vs 三路融合" 相当于在一个受控环境中做 A/B 对比，改进的绝对提升量、相对提升比例、跨指标的一致性都具有可解释的因果意义。

也正因为基线被限定为 Qwen-VL-Max，本文的结果**不能直接与 OSrCIR 原论文的绝对数值比较**（GPT-4o 基线的起点更高，改进空间可能更小或更大）。如果后续工作能将本文方法迁移到 GPT-4o 基线上，验证改进是否同样成立，将是一个有价值的扩展实验方向。

表 [4-3](#tab-fiq) 中 Ensemble 一列是仅做描述融合（$D_1/D_2$，即 $\alpha=1.0$ 的退化方案）的结果，可与最终三路融合一列对比。观察到两个规律：其一，Ensemble 相比 Baseline 已带来显著提升（dress R@10 +2.50、shirt +1.35、toptee +3.48），说明 $D_1$ 与 $D_2$ 的特征融合本身就是有效的，V7 反幻觉精炼的价值在 Ensemble 阶段已被验证；其二，加入代理图信号的三路融合在此基础上又有进一步增益（dress R@10 从 18.30 升到 19.29），表明 CLIP($P$) 与文本融合特征 $\mathbf{f}_{\text{text}}$ 之间仍存在可利用的信息差，二者并非冗余。

dress 子集的提升幅度最大（R@10 +3.49，相对 +22.1%），原因可能是连衣裙的修改类型更丰富（长度、袖型、领口、腰身等属性互相独立），MLLM 初始描述更容易遗漏其中某个维度，代理图的视觉对照因此能发挥更大作用。shirt 子集提升最小（R@10 +1.35，相对 +5.2%），其原因一方面是 shirt 的修改文本更集中于颜色和图案等 MLLM 较擅长描述的属性，另一方面是 Qwen-VL-Max 在 shirt 上的基线起点本身较高（26.00，比 dress 的 15.80 高出 10pp），进一步改进的空间相对收窄。toptee 的提升（R@10 +4.26，相对 +18.4%）介于两者之间，与直觉一致。

#### 4.2.5 GeneCIS

GeneCIS 是本工作改进幅度最大的数据集，也是对默认参数配置挑战最明显的数据集。其任务设定与前述三者存在本质差异：修改文本通常只有 1–2 个英文单词（如 "focus on: red"、"change object: dog → cat"），每个查询仅对应约 14 张候选图。在这种**短文本 + 微型图库**的极端条件下，一方面 MLLM 在如此少的修改信号上很难准确推理目标描述，另一方面 CLIP 对短文本的编码也容易出现"语义塌缩"——不同短文本被映射到相近特征位置。

在默认参数（$\beta = 0.7$、$\alpha = 0.9$）下，GeneCIS 四个子任务的表现并不理想——其中 focus_object 子集的 R@1 较基线下降 0.87pp。针对该数据集的特性，本文做了两方面适配：**专用 prompt**（进一步缩短精炼描述长度、直接要求 $D_2$ 输出修改后核心语义名词短语而非完整句子）和**任务自适应参数**（对每个子任务独立搜索最优 $\alpha$、$\beta$，详见 4.3.2 节）。经过这两项调整后，最终结果如表 [4-6](#tab-genecis) 所示，4 个子任务的 R@1 全部大幅提升。

**表 4-6** GeneCIS 全量实验结果（专用 prompt + 任务自适应参数）


| 子集               | 样本数  | 指标  | Baseline | 三路融合      | 提升     |
| ---------------- | ---- | --- | -------- | --------- | ------ |
| change_object    | 1960 | R@1 | 13.83    | **25.51** | +11.68 |
| focus_object     | 1960 | R@1 | 16.02    | **23.62** | +7.60  |
| change_attribute | 2110 | R@1 | 12.65    | **21.79** | +9.14  |
| focus_attribute  | 1998 | R@1 | 18.82    | **27.83** | +9.01  |


change_object 子任务提升最为突出（R@1 +11.68，相对 +84.5%），原因在于该子任务的修改语义（物体替换）正是 V7 反幻觉精炼最擅长处理的场景——代理图能够非常清晰地呈现"替换后的新物体"，提供可验证的视觉证据；而默认 MLLM 基线在短文本下容易输出模糊描述（如把 "dog → cat" 写成 "an animal"），代理图的引入则强制 MLLM 在第二轮输出更具体的类别名词。focus_attribute 和 change_attribute 上的提升幅度相当（+47.9% 和 +72.3%），均远高于前三大数据集，说明三路融合在细粒度条件相似度任务上具有更大的发挥空间。focus_object 提升相对最小（+47.4%），但也是从默认参数下 -0.87pp 的退化被翻转为 +7.60pp 的大幅提升，证实了任务自适应参数的必要性。

综合 9 个子任务来看，**基线越弱、图库越小、修改文本越短的任务，三路融合的增益越大**。这与方法机理一致——基线弱意味着 MLLM 推理容易出错，视觉代理的验证作用更有价值；图库小意味着排序压力集中在 R@1 上，方法对头部排序的改善被指标充分放大；修改文本短意味着 MLLM 在第一轮描述上的信息量不足，第二轮精炼和代理图的补充作用更明显。

### 4.3 消融实验与分析

#### 4.3.1 Prompt 版本对比

第 3.3.3 节已详细介绍了 V1–V7 的迭代过程。从实验角度看，该迭代实际构成了一组对"如何使用代理图"的消融研究：V1 代表"直接对比两张图"、V2 代表"对比 + 思维链"、V3 代表"仅用代理图"、V4–V5 代表"加入不可信性声明的渐进尝试"、V6 代表"最大信息量输入"、V7 代表"诊断式使用 + 严格约束"。CIRR 200 样本上 R@10 的轨迹如图 [3-2](#fig-prompt) 所示：V1–V6 全部退化或持平（最低 V6 仅 49.5），V7 是唯一一个显著超越基线（67.5 vs 66.7）的版本。

这一消融结果在技术层面有一个明确结论：**代理图的"使用方式"比代理图本身的质量更关键**。V1–V6 使用的代理图质量和 V7 完全相同（都是同一批 MiniMax 生成结果），但 V1–V6 把代理图当作描述来源时管线整体退化，V7 把代理图降级为诊断工具时管线正向提升。换言之，代理图带来的性能变化是一个**双向性信号**——用对了是增益源，用错了是噪声源，符号由 prompt 决定。

这一观察进一步加强了本文方法的合理性——视觉代理本身不是银弹，关键在于它被整合进管线的方式是否尊重其作为"AI 生成物"的不可靠性。V7 的成功证明，只要通过 prompt 显式切断幻觉传播路径，代理图的正向价值就能被稳定地提取出来。

#### 4.3.2 α / β 参数分析

在 GeneCIS 上，本文对 $\alpha$ 和 $\beta$ 做了密集网格搜索，$\alpha \in \{0.7, 0.75, 0.80, 0.85, 0.90, 0.95, 1.0\}$、$\beta \in \{0.1, 0.3, 0.5, 0.6, 0.7, 0.8, 1.0\}$ 共 49 组组合，对每个子任务各跑一次完整检索并记录 R@1。结果的热力图如图 [4-4](#fig-heatmap) 所示，表 [4-7](#tab-grid) 汇总了各子集的最优参数及其提升效果。

**图 4-4** GeneCIS 三个子集的 $\alpha$ / $\beta$ 网格搜索热力图。颜色表示相对 Baseline 的 R@1 变化量，★ 标记最优参数组合。

**表 4-7** GeneCIS 各子集最优 $\alpha$ / $\beta$ 参数


| 子集               | 默认参数 ΔR@1 | 最优 β | 最优 α | 最优 ΔR@1 |
| ---------------- | --------- | ---- | ---- | ------- |
| change_object    | +0.10     | 0.30 | 0.95 | +1.02   |
| focus_object     | −0.87     | 1.00 | 0.90 | +0.05   |
| change_attribute | +0.28     | 0.80 | 0.80 | +1.33   |
| focus_attribute  | +1.05     | 0.60 | 0.85 | +1.55   |


从最优参数的分布可以读出两类明显规律。第一，**不同子任务对 $\beta$ 的偏好存在显著差异**。change_object 的最优 $\beta = 0.30$，说明 V7 精炼后的 $D_2$ 在该任务中占主导地位——这与直觉一致，物体替换场景下第二轮精炼最能够给出具体的类别名词，比 $D_1$ 的模糊描述更有检索价值。focus_object 的最优 $\beta = 1.00$，意味着该子任务中精炼描述完全无增益，检索质量全部来自 $D_1$——可能因为 focus_object 的修改语义是"聚焦某个物体"而非"改变某物体"，代理图并不能为 $D_2$ 提供有效的视觉差异信号。change_attribute 和 focus_attribute 的最优 $\beta$ 介于 0.6 ~ 0.8 之间，与前三大数据集的默认 $\beta=0.7$ 接近。

第二，**$\alpha$ 的最优值普遍在 0.80 ~ 0.95 区间**，没有出现 $\alpha < 0.70$ 的最优点。这与 FashionIQ/CIRCO/CIRR 上的默认 $\alpha = 0.9$ 基本一致，说明代理图作为"辅助信号而非主导信号"的定位在所有任务上都成立。即使在最适合代理图发挥作用的 change_object 子任务，最优 $\alpha$ 也达到 0.95，代理图权重不超过 5%。这一跨任务一致性为"视觉代理应保持辅助地位"的设计原则提供了坚实的实证支持。

该消融实验同时揭示了方法的一个实际约束——**默认参数并非对所有任务最优**。FashionIQ/CIRCO/CIRR 的最优点与默认值接近（偏差小于 0.1），因此在这些数据集上直接使用默认参数不会损失太多性能；但 GeneCIS 各子任务的最优参数差异较大，不做任务自适应调整会损失 1.0 ~ 8.5pp 不等。这一现象对方法的工程落地具有实际意义：对于任务结构相近的一组数据集，可以共享一套参数；对于短文本/小图库类的新场景，则建议事先做一次网格搜索。

#### 4.3.3 典型案例分析

为更直观地展示三路融合各环节的作用，本文从 CIRCO、CIRR、FashionIQ dress 各抽取一个典型正向案例进行定性分析。

**案例 1（CIRR）：** 参考图为一只白色小狗坐在草地上，修改文本 "change the dog to an orange cat"。基线 $D_1$ 输出 "a cat in the grass"，遗漏了颜色修饰；目标图像在基线下排名第 14。代理图正确地画出了橙色猫在草地上的场景；V7 精炼后的 $D_2$ 为 "an orange cat sitting in green grass"，补齐了颜色。三路融合后目标图像排名升至第 2。

**案例 2（CIRCO）：** 参考图为餐厅内的一张桌子，修改文本 "the same scene but without the flowers"。基线 $D_1$ 输出 "a restaurant table"，未体现"去除花朵"这一关键修改；目标图像在基线下排名 47，远离 Top 10。代理图画出了一张没有花的餐厅桌子，与 $D_1$ 的字面表达匹配；V7 精炼后的 $D_2$ 为 "a restaurant table without flowers or floral decoration"，加入了显式否定。三路融合后目标图像排名升至第 8，成功进入 Top 10。该案例说明，**精炼描述的主要价值在于把隐式语义显式化**，尤其对 CLIP 这种对否定语义不敏感的编码器。

**案例 3（FashionIQ dress）：** 参考图为红色短袖连衣裙，修改文本 "is longer and has floral patterns"。基线 $D_1$ 输出 "a long floral dress"，丢失了原图颜色；目标图为红色碎花长裙。代理图生成了一条不带红色、以蓝色底为主的碎花长裙；V7 精炼时识别出"原图为红色"这一应保留属性未被 $D_1$ 涵盖，最终 $D_2$ 为 "a long red floral dress"。三路融合中，虽然代理图的颜色"错了"（蓝色而非红色），但其 CLIP 图像特征对"长裙 + 碎花"属性的贡献仍帮助目标图像在排名中上升。该案例揭示了一个反直觉现象：**代理图即使存在明显错误，其 CLIP 特征依然可能带来正向贡献**——因为 CLIP 特征编码了代理图的多种属性，错误属性的噪声被其它正确属性的信号部分稀释。

#### 4.3.4 各模块作用演进

从方法结构上看，视觉代理、反幻觉精炼和描述融合各自针对一个具体问题。表 [4-8](#tab-ablation) 按项目实际研发顺序整理了各阶段方案的作用与局限，可视为一个按时间线的"自身消融"记录。

**表 4-8** 方法迭代过程各阶段的作用与局限


| 阶段  | 方案                   | 解决的问题               | 遗留问题          |
| --- | -------------------- | ------------------- | ------------- |
| 1   | Visual Proxy         | $D_1$ 缺乏视觉验证        | 代理图含 AI 幻觉    |
| 2   | 直接精炼                 | 尝试利用代理图修正 $D_1$     | 幻觉反而加剧退化      |
| 3   | V7 反幻觉 prompt        | 切断幻觉引入路径            | 个别子集退化        |

| 5   | 三路融合                 | 统一框架，三信号互补          | GeneCIS 有优化空间 |

表 [4-8](#tab-ablation) 中的演进路径有两个结构性特征值得指出。其一，每个新方案都是在解决上一方案的遗留问题，而不是提出一个全新的独立模块——这种"问题驱动"的迭代保证了方法增长的每一步都有明确的技术依据。其二，方案 3（V7 反幻觉）和方案 4（描述融合）解决的是同一类问题的两个侧面：V7 通过 prompt 设计降低了第二轮精炼本身的幻觉风险，描述融合通过特征加权降低了精炼偶发失误对整体系统的影响；二者互为补充，单独使用都不足以达到最终稳定状态。

### 4.4 方法优势与局限

本文方法的优势主要体现在四个层面。**第一，不依赖目标数据集训练**，保持了零样本方法的完整通用性，任何时候引入新数据集都可以直接使用，无需任何微调流程，这一特性在产品化场景中具有显著价值。**第二，通过视觉代理显式引入图像空间信号**，把原本被锁定在文本空间的管线打开为图文交叉管线，这一结构性改动在本工作之外的 MLLM + CLIP 类管线中同样具有迁移价值。**第三，通过描述融合降低了单次推理失误的系统影响**——FashionIQ dress 上"直接替换 $D_2$"的方案会带来 -0.30pp 的退化，而"融合 $D_1/D_2$"的方案带来 +2.50pp 的增益，差异完全来自特征层面的加权设计，没有改动任何上游组件。**第四，参数化统一接口便于扩展**，$\alpha$、$\beta$ 两个旋钮覆盖了文本/图像、原始/精炼两个维度，使得 GeneCIS 这类特殊任务能通过网格搜索快速找到最优配置。

同时，该方法也存在明确的边界和局限。**其一，推理成本上升**：相比基线每个查询增加一次文生图调用（约 8–12 秒、0.03 元）和一次 MLLM 精炼调用（约 4–7 秒、0.03 元），端到端延迟从 5 秒左右上升到 20 秒左右，单查询 API 费用约翻倍。对于非实时应用（如离线索引构建、报告生成）可接受，但实时在线检索场景可能需要进一步的工程优化（例如代理图离线预生成、精炼描述按需触发）。**其二，参数任务敏感性**：如 4.3.2 节所示，不同任务对 $\alpha$、$\beta$ 的偏好存在显著差异，默认配置并非全局最优。这既是一个局限（需要任务适配）也是一个扩展点（可设计自动化参数预测）。**其三，受 MLLM 上游能力约束**：当 $D_1$ 完全错误时（例如 MLLM 把修改意图完全搞反），代理图会忠实地"画错"，反幻觉精炼也无法从错误的代理图中挽救目标描述。本文方法本质上是一个"补强"框架，其天花板仍然由 MLLM 决定。**其四，文生图模型的领域偏差**：MiniMax image-01 的训练分布偏向互联网常见图像，在特定垂直领域（如医学影像、工业产品）生成的代理图可能与真实数据分布差异过大，使代理图路径在这些领域效果受限；替换为领域专用 T2I 模型是一条自然的扩展路径。

### 4.5 本章小结

本章从实验设置、主实验结果、消融研究、方法讨论四个层面完整呈现了三路融合方法的实证依据。主实验部分在 9 个子任务、35 项指标上取得了全面正向结果，相对提升中位数为 26%，头部指标提升可达 84.5%。消融部分通过 V1–V7 的 prompt 迭代揭示了"如何使用代理图比代理图本身更关键"、通过 GeneCIS 网格搜索揭示了"$\alpha$ 保持辅助地位、$\beta$ 存在任务差异"的规律，并通过三个典型案例给出了各环节的定性解释。方法讨论部分则明确了本方法的四点优势和四点局限，为后续工作提供了清晰的改进方向。本章内容构成了对三路融合方法有效性、通用性与边界条件的完整验证。

## 第 5 章 总结与展望

### 5.1 工作总结

本文围绕零样本组合式图像检索中"推理路径单一、缺乏视觉验证"这一结构性问题展开研究，以 CVPR 2025 的 OSrCIR 为基线，提出并实现了三路融合方法。整个研究包含基线复现、问题诊断、方法设计、提示词迭代、全量实验、消融分析六个阶段，累计使用 MLLM 调用约 14K 次、文生图调用约 6K 次，在 4 个数据集的 9 个子任务、35 项评估指标上完成了完整验证。

在方法层面，本文的主要贡献可概括为三点。**其一，提出视觉代理机制**——把 MLLM 输出的初始描述通过文生图模型投射回图像空间，使原本封闭在文本路径中的推理过程拥有了一个可视化的"中间检查点"，并把该中间产物作为独立检索信号并入管线。**其二，提出反幻觉描述精炼策略**——通过 V1 到 V7 的七轮 prompt 迭代，确立了"代理图只做诊断、不做描述来源"这一核心原则，从机制上切断了文生图幻觉向最终描述的传播路径；V7 是第一个让精炼真正带来稳定增益的 prompt 版本。**其三，提出三路融合框架**——在 CLIP 特征空间分别融合文本两路与图像一路信号，通过 $\alpha$、$\beta$ 两个参数提供可调的证据合并接口，使任一路径的失误都能被其它路径部分抵消，显著提升了系统整体稳健性。

在实验层面，本文的关键结果是**35 个指标全部正向提升，无持平、无退化**。9 个主指标的相对提升区间为 +5.2% ~ +84.5%，全量 35 指标的相对提升中位数为 +26%。这一全面正向性在 ZS-CIR 相关文献中并不多见，且提升幅度与任务特性高度相关——基线越弱、图库越小、修改文本越短、任务越偏细粒度条件判断，三路融合的增益越显著。这一规律本身也是对方法机理的一次验证：视觉代理和描述融合所补充的信号，恰好弥补了 MLLM 在低信号量场景下的固有短板。

此外，本工作在方法论层面得出了一个具有普适性的结论：对于使用 AI 生成图像作为管线中间产物的系统，**如何使用这张图比这张图本身的质量更关键**。V1 到 V6 在代理图质量完全相同的前提下全部退化，V7 仅凭 prompt 设计的改动就实现了正向突破；这一对比强烈暗示，任何涉及"文本→图像→文本"式闭环的推理管线，都应把生成图像的角色严格约束为"诊断工具"而非"描述来源"。该原则不限于组合式图像检索，也可能适用于其他需要利用文生图中间产物的多模态任务。

### 5.2 未来工作展望

在本工作的基础上，至少有以下四个方向值得进一步探索。

**其一，查询自适应参数预测机制。** 4.3.2 节在 GeneCIS 上的网格搜索表明不同子任务的最优 $\alpha$、$\beta$ 差异显著，事先通过网格扫描找参数虽然可行，但对未知新任务不具备直接可迁移性。一个自然的思路是训练一个轻量参数预测器：输入查询的文本长度、候选图库规模、MLLM 初始描述置信度等可离线计算的特征，输出推荐的 $\alpha$、$\beta$，把任务自适应从"人工网格搜索"升级为"查询级别的动态配置"。

**其二，升级上游 MLLM 的验证。** 本文因接口可用性使用 Qwen-VL-Max 作为 MLLM 实现，相比 OSrCIR 原论文的 GPT-4o 基线起点显著偏低（FashionIQ dress R@10 相差 13.9pp）。随着 GPT-4o、Claude Opus 4、Gemini 2.5 等更强 MLLM 的接口逐步可获得，可验证本文方法在 GPT-4o 级别的基线上是否仍能保持正向提升。若改进比例在更强模型上稳定复现，将进一步强化方法的通用性；若改进比例显著收窄，则说明视觉代理主要是在补偿 MLLM 自身能力的不足，其边际价值随模型能力上升而递减，这一观察本身也有指导意义。

**其三，代理图质量优化与多代理图策略。** 本文使用单张代理图作为图像路径输入，但 MiniMax image-01 在某些样本上存在明显质量问题（构图不合理、关键物体被遮挡、颜色失真等）。后续工作可从两个方向改进：一是探索多代理图策略，对同一 $D_1$ 生成多张候选代理图，并通过 CLIP 相似度或质量评分挑选最佳；二是替换或微调更符合目标数据集分布的 T2I 模型（例如针对 FashionIQ 使用服饰专用生成模型），从而缩小代理图与真实候选库之间的领域差异。

**其四，拓展到有监督和其它多模态任务。** 本文框架在零样本设定下已验证其有效性，一个自然的扩展是将其应用到有监督 CIR 任务（如在 FashionIQ/CIRR 训练集上做监督学习）。由于视觉代理和描述融合是后处理式组件，不依赖具体的监督信号，理论上可以直接叠加在任何有监督 CIR 基线之上。此外，该框架的核心思想——"为纯文本管线补充可验证的图像空间信号"——也可能适用于视觉问答、多模态推理、图文匹配等更广泛的任务场景，值得在方法论层面进一步推广。
